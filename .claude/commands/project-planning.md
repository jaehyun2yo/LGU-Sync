# Project Planning

Read and execute the full planning procedure at `.claude/skills/project-planning.md`.

## Quick Reference

1. Status check: `docs/progress.txt` + `docs/features-list.md` + related code
2. Impact analysis: files to change, dependencies, test coverage
3. Write plan: `docs/plans/YYYY-MM-DD-{work-name}.md`
4. Get user approval before any code changes

## Arguments
Pass the feature/task description. If arguments include `--skip-approval`, skip step 4.
