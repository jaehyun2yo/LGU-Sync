# 7. 서킷 브레이커

### 7.1 개요

서킷 브레이커는 LGU+ API 서버가 불안정한 상태일 때 반복적인 실패 요청을 방지하고, 서버가 복구될 시간을 확보하는 보호 패턴이다.

### 7.3 상세 설정값

```typescript
interface CircuitBreakerConfig {
  // 실패율 계산 윈도우 (최근 N개 요청)
  windowSize: 20;

  // 서킷을 열(OPEN)기 위한 실패율 임계값 (%)
  failureThreshold: 40;

  // OPEN 상태 유지 시간 (ms)
  openDurationMs: 30000;

  // HALF_OPEN 상태에서 허용할 테스트 요청 수
  halfOpenRequests: 3;

  // HALF_OPEN에서 CLOSED로 전이하기 위한 최소 성공 수
  halfOpenSuccessThreshold: 2;

  // 서킷 브레이커 모니터링 대상 에러 유형
  monitoredErrors: ['NetworkError', 'ServerError', 'RateLimitError'];

  // AuthError, FileSystemError는 서킷 브레이커 대상 아님
  // (이 에러들은 서버 상태와 무관)
}
```

### 7.5 슬라이딩 윈도우 구현

```typescript
class SlidingWindow {
  private results: Array<{ success: boolean; timestamp: number }>;
  private readonly windowSize: number;

  constructor(windowSize: number) {
    this.results = [];
    this.windowSize = windowSize;
  }

  record(success: boolean): void {
    this.results.push({ success, timestamp: Date.now() });
    // 윈도우 크기 초과 시 가장 오래된 결과 제거
    if (this.results.length > this.windowSize) {
      this.results.shift();
    }
  }

  getFailureRate(): number {
    if (this.results.length === 0) return 0;
    const failures = this.results.filter(r => !r.success).length;
    return (failures / this.results.length) * 100;
  }

  reset(): void {
    this.results = [];
  }
}
```

### 7.6 서킷 브레이커 상태 변경 시 알림/로깅

```typescript
interface CircuitBreakerEvent {
  previousState: 'CLOSED' | 'OPEN' | 'HALF_OPEN';
  currentState: 'CLOSED' | 'OPEN' | 'HALF_OPEN';
  failureRate: number;
  timestamp: number;
  reason: string;
}

// 상태 변경 시 수행되는 작업:
// CLOSED → OPEN:
//   - 경고 로그 기록
//   - GUI에 "API 서버 불안정" 알림 표시
//   - 모든 감지 전략에 일시 중단 요청
//   - 메트릭에 서킷 오픈 이벤트 기록

// OPEN → HALF_OPEN:
//   - 정보 로그 기록
//   - GUI에 "API 서버 복구 확인 중" 알림 표시

// HALF_OPEN → CLOSED:
//   - 정보 로그 기록
//   - GUI에 "API 서버 정상 복구" 알림 표시
//   - 감지 전략 재개
//   - 재시도 큐 처리 시작

// HALF_OPEN → OPEN:
//   - 경고 로그 기록
//   - GUI에 "API 서버 여전히 불안정" 알림 표시
```
