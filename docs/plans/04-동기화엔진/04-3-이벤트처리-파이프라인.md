# 3. 이벤트 처리 파이프라인

### 3.1 이벤트 타입 정의

```typescript
// 이벤트 타입 열거형
enum SyncEventType {
  FILE_CREATED = 'FILE_CREATED',     // 새 파일이 업로드됨
  FILE_MODIFIED = 'FILE_MODIFIED',   // 기존 파일이 수정됨 (같은 경로, 다른 내용)
  FILE_DELETED = 'FILE_DELETED',     // 파일이 삭제됨
  FOLDER_CREATED = 'FOLDER_CREATED', // 새 폴더가 생성됨
  FOLDER_DELETED = 'FOLDER_DELETED', // 폴더가 삭제됨 (하위 파일 포함)
}

// 이벤트 인터페이스
interface SyncEvent {
  id: string;                    // UUID v7 (시간 순서 보장)
  type: SyncEventType;           // 이벤트 타입
  source: 'primary' | 'secondary' | 'tertiary';  // 감지 전략
  path: string;                  // LGU+ 웹하드 상의 전체 경로
  fileName: string;              // 파일명
  fileSize: number;              // 파일 크기 (bytes)
  mtime: number;                 // 수정 시각 (Unix timestamp ms)
  parentPath: string;            // 부모 폴더 경로
  fileKey: string;               // 중복 제거용 키
  priority: number;              // 처리 우선순위 (낮을수록 높음)
  createdAt: number;             // 이벤트 생성 시각
  retryCount: number;            // 재시도 횟수
  metadata: Record<string, unknown>;  // 추가 메타데이터
}

// 이벤트 처리 결과
interface SyncEventResult {
  eventId: string;
  status: 'completed' | 'failed' | 'skipped';
  error?: string;
  downloadedBytes?: number;
  uploadedBytes?: number;
  durationMs: number;
  completedAt: number;
}
```

### 3.2 이벤트 필터링 규칙

감지된 이벤트 중 처리가 불필요한 이벤트를 걸러내는 필터링 단계이다.

```
┌──────────────────────────────────────────────────────────┐
│                이벤트 필터링 파이프라인                     │
└──────────────────────────────────────────────────────────┘

  원시 이벤트
       │
       ▼
  ┌─────────────────────────┐
  │ 1. 경로 필터             │  "올리기전용/" 하위만 허용
  │    허용: 올리기전용/**   │  "내리기전용/", 기타 → 폐기
  └────────┬────────────────┘
           │ 통과
           ▼
  ┌─────────────────────────┐
  │ 2. 파일 확장자 필터       │  .tmp, .partial, .crdownload → 폐기
  │    제외: 임시 파일       │  (다운로드 중인 임시 파일)
  └────────┬────────────────┘
           │ 통과
           ▼
  ┌─────────────────────────┐
  │ 3. 파일명 패턴 필터       │  ~$*, .~lock.*, Thumbs.db → 폐기
  │    제외: 시스템 파일     │  desktop.ini, .DS_Store → 폐기
  └────────┬────────────────┘
           │ 통과
           ▼
  ┌─────────────────────────┐
  │ 4. 파일 크기 필터         │  0 bytes → 폐기 (빈 파일)
  │    제외: 빈 파일         │  > 10GB → 경고 + 수동 처리
  └────────┬────────────────┘
           │ 통과
           ▼
  ┌─────────────────────────┐
  │ 5. 중복 제거             │  fileKey가 이미 처리된
  │    제외: 이미 처리됨     │  이벤트면 폐기
  └────────┬────────────────┘
           │ 통과
           ▼
  유효한 이벤트 → 큐잉
```

**필터 설정 (구성 가능):**

```typescript
interface FilterConfig {
  // 허용할 루트 경로
  allowedRootPaths: string[];       // ['/올리기전용']

  // 제외할 파일 확장자
  excludeExtensions: string[];      // ['.tmp', '.partial', '.crdownload', '.lock']

  // 제외할 파일명 패턴 (glob)
  excludePatterns: string[];        // ['~$*', '.~lock.*', 'Thumbs.db', 'desktop.ini', '.DS_Store']

  // 최소 파일 크기 (bytes)
  minFileSize: number;              // 1 (0바이트 파일 제외)

  // 최대 파일 크기 (bytes, 초과 시 경고)
  maxFileSize: number;              // 10 * 1024 * 1024 * 1024 (10GB)

  // 사용자 정의 필터 함수 (확장용)
  customFilters?: ((event: SyncEvent) => boolean)[];
}
```

### 3.3 이벤트 우선순위

이벤트는 타입에 따라 우선순위가 부여되며, 우선순위가 높은 이벤트가 먼저 처리된다.

```
  우선순위 (숫자가 낮을수록 높은 우선순위)

  ┌──────────────────────────────────────────────────┐
  │  1 (최고)  │  FOLDER_CREATED                     │
  │            │  폴더가 먼저 생성되어야 파일을 넣을   │
  │            │  수 있으므로 최우선                   │
  ├────────────┼─────────────────────────────────────┤
  │  2         │  FILE_CREATED                       │
  │            │  새 파일 동기화가 핵심 기능           │
  ├────────────┼─────────────────────────────────────┤
  │  3         │  FILE_MODIFIED                      │
  │            │  기존 파일 업데이트                   │
  ├────────────┼─────────────────────────────────────┤
  │  4         │  FOLDER_DELETED                     │
  │            │  폴더 삭제는 하위 파일 삭제 후 처리   │
  ├────────────┼─────────────────────────────────────┤
  │  5 (최저)  │  FILE_DELETED                       │
  │            │  삭제는 가장 낮은 우선순위            │
  └──────────────────────────────────────────────────┘
```

**동일 우선순위 내 정렬:**

1. 이벤트 생성 시각 (`createdAt`) 오름차순 (먼저 감지된 것 먼저)
2. 재시도 횟수 (`retryCount`) 오름차순 (처음 시도하는 것 먼저)
3. 파일 크기 (`fileSize`) 오름차순 (작은 파일 먼저 → 빠른 처리)

### 3.4 이벤트 배치 처리

여러 이벤트를 묶어서 효율적으로 처리하는 배치 처리 전략이다.

```
┌──────────────────────────────────────────────────────────┐
│                  이벤트 배치 처리 흐름                     │
└──────────────────────────────────────────────────────────┘

  이벤트 큐
  ┌─────────────────────────────────────────────┐
  │ [E1] [E2] [E3] [E4] [E5] [E6] [E7] [E8]   │
  └─────────────────────────────────────────────┘
       │
       ▼
  ┌─────────────────────────────────────┐
  │ 배치 그룹 생성 조건:                 │
  │                                     │
  │ 1. 같은 부모 폴더의 이벤트 묶기      │
  │ 2. 최대 배치 크기: 10개              │
  │ 3. 최대 대기 시간: 2초               │
  │    (2초 내에 큐에 추가된 것만 묶기)   │
  └─────────┬───────────────────────────┘
            │
            ▼
  ┌──────────────────────────────────────────┐
  │ 배치 1: /올리기전용/업체A/               │
  │   [E1: file1.dxf] [E2: file2.dxf]       │
  │                                          │
  │ 배치 2: /올리기전용/업체B/               │
  │   [E3: file3.pdf] [E4: file4.ai]        │
  │                                          │
  │ 배치 3: /올리기전용/업체C/               │
  │   [E5: file5.dxf]                       │
  └──────────────────────────────────────────┘
```

**배치 처리의 장점:**

1. 같은 폴더의 파일을 연속 처리하여 API 인증 재사용
2. 폴더 생성을 한 번만 수행 (중복 mkdir 방지)
3. 내부 웹하드 업로드 시 같은 경로의 파일을 묶어서 전송 가능

```typescript
interface BatchConfig {
  maxBatchSize: 10;        // 한 배치의 최대 이벤트 수
  maxWaitMs: 2000;         // 배치 수집 최대 대기 시간
  groupBy: 'parentPath';   // 그룹핑 기준
}
```
