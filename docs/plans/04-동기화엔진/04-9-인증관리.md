# 9. 인증 관리

### 9.1 LGU+ 쿠키 기반 인증 흐름

LGU+ 웹하드는 공식 API를 제공하지 않으므로, 웹 브라우저의 쿠키 기반 인증을 역공학하여 사용한다.

```
┌──────────────────────────────────────────────────────────────┐
│                 LGU+ 인증 흐름 전체도                          │
└──────────────────────────────────────────────────────────────┘

  ┌────────────────────┐
  │ 저장된 쿠키 확인    │
  │ (cookie-store.json)│
  └────────┬───────────┘
           │
     ┌─────┴─────┐
     │           │
  쿠키 있음    쿠키 없음
     │           │
     ▼           │
  ┌──────────────┐   │
  │ 쿠키 유효성   │   │
  │ 검증          │   │
  │ (API 호출)   │   │
  └────┬─────────┘   │
       │              │
   ┌───┴───┐         │
   │       │         │
  유효   만료/무효    │
   │       │         │
   │       ├─────────┘
   │       │
   │       ▼
   │  ┌──────────────────────────┐
   │  │ Playwright 로그인 자동화  │
   │  │                          │
   │  │ 1. 브라우저 실행 (headless)│
   │  │ 2. 로그인 페이지 이동     │
   │  │ 3. ID/PW 입력             │
   │  │ 4. 로그인 버튼 클릭       │
   │  │ 5. 로그인 성공 확인       │
   │  │ 6. 쿠키 추출              │
   │  │ 7. 브라우저 종료          │
   │  └────────┬─────────────────┘
   │           │
   │     ┌─────┴─────┐
   │     │           │
   │  로그인 성공   로그인 실패
   │     │           │
   │     ▼           ▼
   │  쿠키 저장    사용자 알림
   │     │        (ID/PW 확인 필요)
   │     │
   ├─────┘
   │
   ▼
  ┌──────────────────────────┐
  │ 쿠키를 API 클라이언트에    │
  │ 주입하여 인증된 요청 수행  │
  └──────────────────────────┘
```

### 9.2 Playwright 사용한 로그인 자동화

```typescript
interface PlaywrightLoginConfig {
  // 로그인 페이지 URL
  loginUrl: 'https://only.webhard.co.kr/login';

  // 브라우저 모드
  headless: true;

  // 로그인 타임아웃
  loginTimeoutMs: 30000;

  // 로그인 성공 판별 기준
  successIndicator: {
    type: 'url_change';     // URL 변경으로 판별
    expectedUrl: '/main';    // 로그인 성공 시 리다이렉트되는 URL
  };

  // 브라우저 사용자 에이전트
  userAgent: 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...';
}
```

**로그인 자동화 상세 단계:**

```
  1. Playwright Chromium 브라우저 실행 (headless 모드)
  2. 로그인 페이지 이동 (https://only.webhard.co.kr/login)
  3. 페이지 로드 완료 대기 (networkidle)
  4. ID 입력 필드 선택 → 사용자 ID 입력
  5. PW 입력 필드 선택 → 사용자 비밀번호 입력
  6. 로그인 버튼 클릭
  7. 페이지 이동 대기 (URL 변경 또는 특정 요소 출현)
  8. 로그인 성공 확인:
     - URL이 /main으로 변경되었는가?
     - 에러 메시지가 표시되지 않았는가?
  9. 성공 시: 브라우저 컨텍스트에서 모든 쿠키 추출
  10. 쿠키를 cookie-store.json에 저장
  11. 브라우저 종료
```

### 9.4 쿠키 저장/복원 (파일 기반)

```typescript
interface CookieStore {
  // 쿠키 저장 파일 경로
  filePath: './data/cookie-store.json';

  // 쿠키 암호화 (OS의 키체인 또는 DPAPI 사용)
  encrypted: true;

  // 저장되는 쿠키 정보
  cookies: Array<{
    name: string;          // 쿠키 이름
    value: string;         // 쿠키 값 (암호화됨)
    domain: string;        // 도메인
    path: string;          // 경로
    expires: number;       // 만료 시각 (Unix timestamp)
    httpOnly: boolean;
    secure: boolean;
    sameSite: 'Strict' | 'Lax' | 'None';
  }>;

  // 메타데이터
  savedAt: number;           // 저장 시각
  loginMethod: 'playwright'; // 로그인 방법
}
```

**쿠키 저장 시점:**

1. 최초 로그인 성공 시
2. 재인증 성공 시
3. API 요청 후 새 쿠키가 Set-Cookie 헤더로 전달되었을 때

**쿠키 복원 시점:**

1. 프로그램 시작 시
2. API 클라이언트 초기화 시

### 9.5 인증 실패 시 사용자 알림

```typescript
interface AuthFailureNotification {
  // GUI 알림 타입별 메시지
  messages: {
    cookieExpired: '세션이 만료되었습니다. 자동 재로그인을 시도합니다.';
    reauthFailed: '자동 재로그인에 실패했습니다. 로그인 정보를 확인해주세요.';
    invalidCredentials: '아이디 또는 비밀번호가 올바르지 않습니다.';
    captchaRequired: '캡차 인증이 필요합니다. 수동으로 로그인해주세요.';
    accountLocked: '계정이 잠겼습니다. LGU+ 웹하드 고객센터에 문의해주세요.';
  };

  // 알림 동작
  actions: {
    reauthFailed: 'show_login_dialog';     // 수동 로그인 다이얼로그 표시
    captchaRequired: 'show_browser';        // 실제 브라우저 창을 열어 수동 로그인
    accountLocked: 'show_message_only';     // 메시지만 표시
  };
}
```
