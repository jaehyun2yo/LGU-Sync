# 11. 성능 최적화

### 11.1 HTTP/2 멀티플렉싱 (undici)

Node.js의 기본 HTTP 클라이언트 대신 undici를 사용하여 HTTP/2 멀티플렉싱과 커넥션 풀링의 이점을 활용한다.

```typescript
import { Pool, request } from 'undici';

interface HttpClientConfig {
  // 커넥션 풀 설정
  pool: {
    // LGU+ API용 풀
    lguplusPool: {
      origin: 'https://only.webhard.co.kr';
      connections: 6;          // 최대 동시 커넥션 수
      pipelining: 1;           // HTTP 파이프라이닝 깊이
      keepAliveTimeout: 30000; // Keep-Alive 타임아웃 (30초)
      keepAliveMaxTimeout: 60000;
    };

    // 내부 웹하드용 풀 (R2/API)
    internalPool: {
      origin: 'https://api.yjlaser.com';
      connections: 4;
      pipelining: 1;
      keepAliveTimeout: 30000;
    };
  };

  // 요청 타임아웃
  headersTimeout: 10000;       // 헤더 수신 타임아웃 (10초)
  bodyTimeout: 300000;         // 바디 수신 타임아웃 (5분, 대용량 파일용)

  // 자동 재시도 (undici 레벨)
  maxRetries: 0;               // undici 레벨 재시도 비활성화
                               // (애플리케이션 레벨에서 재시도 관리)
}
```

**undici 선택 이유:**

| 기능 | node:http | undici |
|------|-----------|-------|
| HTTP/2 지원 | 제한적 | 네이티브 |
| 커넥션 풀링 | 수동 관리 | 내장 |
| 스트리밍 | 지원 | 최적화됨 |
| 성능 | 기본 | 2-3배 빠름 |
| 메모리 | 기본 | 최적화됨 |

### 11.3 배치 처리

여러 작업을 묶어서 처리하여 오버헤드를 줄인다.

```typescript
interface BatchProcessingConfig {
  // DB 쓰기 배치
  dbWriteBatch: {
    maxSize: 50;             // 최대 50건씩 배치 INSERT
    maxWaitMs: 1000;         // 최대 1초 대기 후 플러시
  };

  // 이벤트 처리 배치
  eventBatch: {
    maxSize: 10;             // 최대 10개 이벤트 배치
    maxWaitMs: 2000;         // 최대 2초 대기
    groupBy: 'parentPath';   // 부모 폴더별 그룹핑
  };

  // 체크포인트 플러시 배치
  checkpointBatch: {
    maxWaitMs: 30000;        // 30초 간격 플러시
  };
}
```

### 11.4 메모리 관리 (대용량 파일)

```
┌──────────────────────────────────────────────────────────────┐
│                    메모리 관리 전략                             │
└──────────────────────────────────────────────────────────────┘

  ┌───────────────────────────────────────────────────┐
  │ 1. 스트림 기반 파일 처리                            │
  │                                                   │
  │    파일 크기에 관계없이 메모리 사용량 일정 (~64KB)   │
  │    ReadableStream → TransformStream → WriteStream │
  │    버퍼 크기: highWaterMark = 64KB                 │
  └───────────────────────────────────────────────────┘

  ┌───────────────────────────────────────────────────┐
  │ 2. 스냅샷 메모리 최적화                             │
  │                                                   │
  │    파일 10,000개 기준 스냅샷 메모리: ~5MB           │
  │    - 각 SnapshotEntry: ~500 bytes                 │
  │    - Map 오버헤드 포함                             │
  │                                                   │
  │    이전 스냅샷은 비교 완료 후 즉시 GC 대상          │
  │    (참조 해제)                                     │
  └───────────────────────────────────────────────────┘

  ┌───────────────────────────────────────────────────┐
  │ 3. LRU 캐시 크기 제한                              │
  │                                                   │
  │    중복 제거 캐시: 최대 10,000건                    │
  │    - 각 엔트리: ~100 bytes                        │
  │    - 최대 메모리: ~1MB                            │
  │    - 초과 시 가장 오래된 항목 제거                  │
  └───────────────────────────────────────────────────┘

  ┌───────────────────────────────────────────────────┐
  │ 4. 이벤트 큐 메모리 제한                            │
  │                                                   │
  │    메모리 큐 최대: 1,000건                         │
  │    초과 시: DB 기반 큐로 오버플로                   │
  │    - 메모리 큐 처리 → DB 큐에서 보충               │
  └───────────────────────────────────────────────────┘
```

**메모리 모니터링:**

```typescript
// 프로세스 메모리 사용량 주기적 기록 (1분 간격)
setInterval(() => {
  const usage = process.memoryUsage();
  logger.debug('메모리 사용량', {
    heapUsed: `${Math.round(usage.heapUsed / 1024 / 1024)}MB`,
    heapTotal: `${Math.round(usage.heapTotal / 1024 / 1024)}MB`,
    rss: `${Math.round(usage.rss / 1024 / 1024)}MB`,
    external: `${Math.round(usage.external / 1024 / 1024)}MB`,
  });

  // 힙 사용량이 512MB 초과 시 경고
  if (usage.heapUsed > 512 * 1024 * 1024) {
    logger.warn('메모리 사용량 경고: 힙 512MB 초과');
    // 강제 GC 요청 (--expose-gc 옵션 사용 시)
    if (global.gc) global.gc();
  }
}, 60000);
```
