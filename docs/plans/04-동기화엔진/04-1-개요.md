## 1. 동기화 엔진 개요

### 1.1 엔진의 역할과 책임

동기화 엔진은 외부웹하드동기화프로그램 v2의 핵심 컴포넌트로서, LGU+ 웹하드(only.webhard.co.kr)의 `올리기전용/` 폴더에 거래처가 업로드한 파일을 감지하여 내부 웹하드(Cloudflare R2 기반)로 자동 동기화하는 단방향(one-way) 동기화를 담당한다.

**핵심 책임:**

| 책임 | 설명 |
|------|------|
| 변경 감지 | LGU+ 웹하드의 파일 변경사항을 100% 누락 없이 감지 |
| 이벤트 처리 | 감지된 변경사항을 이벤트로 변환하고 우선순위에 따라 처리 |
| 파일 전송 | LGU+ 웹하드에서 파일을 다운로드하여 내부 웹하드로 업로드 |
| 무결성 보장 | 전송된 파일의 무결성을 검증하고 누락 방지 |
| 오류 복구 | 네트워크 오류, 인증 만료 등 장애 상황에서 자동 복구 |
| 상태 관리 | 동기화 진행 상태를 체크포인트로 기록하여 재시작 시 이어서 처리 |
| 인증 유지 | LGU+ 쿠키 기반 인증 세션을 자동 갱신 |

**v1 대비 v2의 핵심 개선점:**

- v1 문제: 간헐적 감지 누락 (폴링 단일 전략 의존)
- v2 해결: 다중 감지 전략 (폴링 + 스냅샷 비교 + 무결성 검증) 으로 100% 감지 보장

**설계 원칙:**

1. **감지 무결성**: 어떤 파일도 누락되어서는 안 된다 (최소 1회 감지 보장)
2. **멱등성**: 같은 이벤트를 여러 번 처리해도 결과가 동일해야 한다
3. **내결함성**: 일시적 장애에서 자동 복구되어야 한다
4. **관찰 가능성**: 모든 동작이 로깅되고 모니터링 가능해야 한다
5. **단순성**: 단일 사용자 데스크톱 앱에 맞는 적절한 복잡도 유지

### 1.2 동기화 파이프라인 전체 흐름도

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        동기화 파이프라인 전체 흐름                         │
└─────────────────────────────────────────────────────────────────────────┘

  LGU+ 웹하드                                              내부 웹하드
  (only.webhard.co.kr)                                    (Cloudflare R2)
       │                                                        ▲
       ▼                                                        │
  ┌─────────┐    ┌──────────┐    ┌────────┐    ┌──────────┐    ┌─────────┐
  │  감 지   │───▶│  필터링   │───▶│  큐잉   │───▶│ 다운로드  │───▶│  업로드  │
  │ Detect  │    │  Filter  │    │ Queue  │    │ Download │    │ Upload  │
  └─────────┘    └──────────┘    └────────┘    └──────────┘    └─────────┘
       │              │              │              │               │
       │              │              │              │               ▼
       │              │              │              │          ┌─────────┐
       │              │              │              │          │  검 증   │
       │              │              │              │          │ Verify  │
       │              │              │              │          └─────────┘
       │              │              │              │               │
       ▼              ▼              ▼              ▼               ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │                       기 록 (Record to DB)                          │
  │                   SQLite - sync_events 테이블                       │
  └──────────────────────────────────────────────────────────────────────┘
```

**각 단계 상세:**

| 단계 | 입력 | 출력 | 실패 시 |
|------|------|------|---------|
| 감지 (Detect) | LGU+ API 응답 | 원시 변경 이벤트 | 다음 주기에 재감지 |
| 필터링 (Filter) | 원시 이벤트 | 유효한 이벤트 | 이벤트 폐기 + 로그 |
| 큐잉 (Queue) | 유효한 이벤트 | 우선순위 정렬된 큐 | 메모리 큐 → DB 백업 |
| 다운로드 (Download) | 큐의 이벤트 | 로컬 임시 파일 | 재시도 큐 이동 |
| 업로드 (Upload) | 로컬 파일 | 내부 웹하드 파일 | 재시도 큐 이동 |
| 검증 (Verify) | 업로드 결과 | 검증 완료 상태 | 재다운로드/업로드 |
| 기록 (Record) | 검증 결과 | DB 레코드 | WAL 모드로 보장 |

### 1.3 엔진 생명주기

동기화 엔진은 다음의 생명주기 상태를 가진다:

```
                    ┌──────────────────────────────┐
                    │        엔진 생명주기           │
                    └──────────────────────────────┘

                         initialize()
                              │
                              ▼
                    ┌──────────────────┐
                    │   INITIALIZING   │
                    │   (초기화 중)     │
                    └────────┬─────────┘
                             │ 인증 성공 + DB 준비 완료
                             ▼
              ┌──────────────────────────────┐
     ┌───────▶│           RUNNING            │◀──────────┐
     │        │          (실행 중)            │           │
     │        └──────┬────────────┬──────────┘           │
     │               │            │                      │
     │     pause()   │            │  stop()      resume()│
     │               ▼            ▼                      │
     │   ┌────────────────┐  ┌──────────────┐            │
     │   │    PAUSED      │  │  STOPPING    │            │
     │   │  (일시정지)     │  │  (종료 중)   │            │
     │   └───────┬────────┘  └──────┬───────┘            │
     │           │                  │                    │
     │   resume()│                  │ 모든 작업 완료      │
     │           │                  ▼                    │
     │           │         ┌──────────────┐              │
     │           └────────▶│   STOPPED    │              │
     │                     │   (중지됨)    │              │
     │                     └──────┬───────┘              │
     │                            │                      │
     │                    start() │                      │
     │                            └──────────────────────┘
     │
     │  에러 발생 시 자동 복구 시도
     │         ┌──────────────┐
     └─────────│    ERROR     │
               │  (오류 상태)  │
               └──────────────┘
                     │ 복구 불가 시
                     ▼
              ┌──────────────┐
              │    FATAL     │
              │ (치명적 오류) │
              └──────────────┘
```

**각 상태 설명:**

| 상태 | 설명 | 허용되는 전이 |
|------|------|--------------|
| INITIALIZING | 엔진 초기화 중. DB 연결, 인증, 체크포인트 복원 수행 | → RUNNING, → ERROR |
| RUNNING | 정상 동작 중. 모든 감지 전략이 스케줄에 따라 실행됨 | → PAUSED, → STOPPING, → ERROR |
| PAUSED | 일시 정지. 진행 중인 다운로드/업로드는 완료하지만 새 감지는 중단 | → RUNNING, → STOPPING |
| STOPPING | 종료 진행 중. 진행 중인 작업 완료를 기다리고 체크포인트 저장 | → STOPPED |
| STOPPED | 완전 중지. 모든 리소스 해제됨 | → RUNNING (재시작) |
| ERROR | 오류 발생. 자동 복구를 시도하며, 복구 성공 시 RUNNING으로 전이 | → RUNNING, → FATAL |
| FATAL | 복구 불가능한 치명적 오류. 사용자 개입 필요 | → STOPPED (수동) |

**상태 전이 시 수행되는 작업:**

```typescript
// 상태 전이 훅 (의사코드)
interface LifecycleHooks {
  onInitializing(): Promise<void>;  // DB 마이그레이션, 인증, 체크포인트 복원
  onRunning(): Promise<void>;       // 감지 스케줄러 시작, 큐 프로세서 시작
  onPausing(): Promise<void>;       // 새 감지 중단, 진행 중 작업 완료 대기
  onResuming(): Promise<void>;      // 감지 스케줄러 재시작, 놓친 이벤트 확인
  onStopping(): Promise<void>;      // 모든 타이머 해제, 큐 비우기, 체크포인트 저장
  onStopped(): Promise<void>;       // DB 연결 종료, 리소스 해제
  onError(error: Error): Promise<void>;  // 에러 로깅, 복구 시도
}
```

### 1.4 SDD 레이어 매핑

본 문서의 각 섹션은 SDD 스펙 레벨에 대응한다:

| 문서 섹션 | SDD Level | 스펙 유형 | 설명 |
|---|---|---|---|
| §1 엔진 개요 | — | 아키텍처 문서 | 설계 배경 및 구조 |
| §2 변경 감지 전략 | Level 2 | 인터페이스 스펙 | `IFileDetector` 계약 |
| §3 이벤트 처리 파이프라인 | Level 2 | 인터페이스 스펙 | `IEventBus`, 파이프라인 단계 |
| §4 다운로드 관리 | Level 2 | 인터페이스 스펙 | `IDownloadManager` 계약 |
| §5 업로드 관리 | Level 2 | 인터페이스 스펙 | `IWebhardUploader` 계약 |
| §6 재시도 전략 | Level 2 | 인터페이스 스펙 | `IRetryManager` 계약 |
| §7 서킷 브레이커 | Level 2 | 인터페이스 스펙 | `ICircuitBreaker` 계약 |
| §8 체크포인트 시스템 | Level 1 | 타입 스펙 | 체크포인트 데이터 구조 |
| §9 인증 관리 | Level 2 | 인터페이스 스펙 | 인증 세션 관리 |
| §10 동시성 제어 | Level 2 | 인터페이스 스펙 | 세마포어/큐 계약 |
| §11 성능 최적화 | Level 1 | 설정 스펙 | 설정값/임계치 정의 |
| §12 모니터링 | Level 1 | 타입 스펙 | 메트릭 데이터 구조 |
