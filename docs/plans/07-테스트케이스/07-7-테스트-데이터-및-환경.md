## 7. 테스트 데이터 및 환경

### 7.1 Mock 서버 구성

```
tests/
├── mocks/
│   ├── msw-handlers/
│   │   ├── lguplus-handlers.ts       # LGU+ API Mock 핸들러
│   │   ├── webhard-handlers.ts       # 자체웹하드 API Mock 핸들러
│   │   └── index.ts                  # 통합 설정
│   ├── fixtures/
│   │   ├── history-response.json     # 히스토리 API 응답 샘플
│   │   ├── folder-list-response.json # 폴더 목록 응답 샘플
│   │   ├── login-response.json       # 로그인 응답 샘플
│   │   └── download-info.json        # 다운로드 정보 응답 샘플
│   ├── files/
│   │   ├── sample-1kb.dxf            # 테스트용 DXF (1KB)
│   │   ├── sample-10mb.bin           # 대용량 테스트 (10MB)
│   │   └── sample-special-chars.dxf  # 특수문자 파일명
│   └── mock-uploader.ts              # IWebhardUploader Mock 구현체
├── helpers/
│   ├── db-helper.ts                  # 인메모리 SQLite 헬퍼
│   ├── ipc-mock.ts                   # IPC invoke/send Mock
│   └── test-factory.ts               # SyncEvent, 설정 등 팩토리
└── setup.ts                          # Vitest 글로벌 설정
```

### 7.2 Mock 핸들러 상세

| 엔드포인트 | Mock 동작 | 용도 |
|-----------|----------|------|
| `POST /login-process` | 성공: 쿠키 세트, 실패: 403 | 로그인 테스트 |
| `GET /api/history` | since_no 기반 필터 응답 | 폴링 감지 테스트 |
| `GET /api/folders` | 정적 폴더 트리 반환 | 스냅샷 비교 테스트 |
| `GET /downloads/:id/server` | session/nonce/url 반환 | 다운로드 정보 테스트 |
| `GET /download-file` | 바이너리 스트림 반환 | 파일 다운로드 테스트 |
| `POST /api/upload` | 성공: etag 반환 | 업로드 테스트 |
| `HEAD /api/files/:path` | 존재: 200, 미존재: 404 | 무결성 검증 테스트 |

### 7.3 테스트 데이터 세트

| 데이터 세트 | 내용 | 용도 |
|------------|------|------|
| **기본 폴더 구조** | 올리기전용/ 하위 업체A~C, 각 2~5파일 | 대부분의 테스트 |
| **대량 파일** | 100개 파일 (히스토리 100건) | 대량 처리 테스트 |
| **특수문자 파일** | `[어뮤즈][ONE-26-54] 가차_목형.DXF` 등 | 파일명 처리 테스트 |
| **대용량 파일** | 10MB, 100MB Mock 스트림 | 스트림/타임아웃 테스트 |
| **에러 응답** | 401, 403, 404, 429, 500 응답 | 에러 처리 테스트 |
| **손상된 DB** | 일부 테이블 누락/손상 DB 파일 | DB 복구 테스트 |

### 7.5 테스트 실행 명령

| 명령 | 대상 | 설명 |
|------|------|------|
| `pnpm test:core` | Core 단위 테스트 | Vitest (node 환경) |
| `pnpm test:main` | Main 통합 테스트 | Vitest (node 환경, Electron Mock) |
| `pnpm test:ui` | Renderer 컴포넌트 | Vitest (jsdom 환경, RTL) |
| `pnpm test:e2e` | E2E 시나리오 | Playwright (Electron 실행) |
| `pnpm test` | 전체 테스트 | Core + Main + UI |
| `pnpm test:coverage` | 커버리지 리포트 | 전체 + HTML 리포트 생성 |
