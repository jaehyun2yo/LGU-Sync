## 테스트-스펙 추적 매트릭스

테스트 케이스와 SDD 인터페이스 스펙의 추적 관계:

### Core 레이어

| TC ID 범위 | 인터페이스 | 메서드 | 테스트 수 |
|---|---|---|---|
| TC-CORE-SYNC-* | `ISyncEngine` | `start`, `stop`, `pause`, `resume`, `fullSync` | §2.1 참조 |
| TC-CORE-DET-* | `IFileDetector` | `startPolling`, `stopPolling`, `detectChanges` | §2.2 참조 |
| TC-CORE-LGU-* | `ILGUplusClient` | `login`, `downloadFile`, `getUploadHistory` | §2.3 참조 |
| TC-CORE-RETRY-* | `IRetryManager` | `execute`, `getRetryDelay`, `shouldRetry` | §2.4 참조 |
| TC-CORE-STATE-* | `IStateManager` | `getState`, `setState`, `transition` | §2.5 참조 |
| TC-CORE-EVT-* | `IEventBus` | `emit`, `on`, `off`, `once` | §2.6 참조 |
| TC-CORE-DL-* | `IDownloadManager` | `download`, `batchDownload`, `cancel` | §2.7 참조 |
| TC-CORE-CFG-* | `IConfigManager` | `get`, `set`, `getAll`, `validate` | §2.8 참조 |

### IPC / Main 레이어

| TC ID 범위 | 스펙 참조 | 설명 |
|---|---|---|
| TC-MAIN-IPC-* | `IpcHandlers` 타입 맵 | IPC 핸들러 계약 테스트 |
| TC-MAIN-TRAY-* | Electron Tray API | 시스템 트레이 동작 |
| TC-MAIN-AUTO-* | `AppConfig.autoStart` | 자동 시작 설정 |

### Renderer 레이어

| TC ID 범위 | 스펙 참조 | 설명 |
|---|---|---|
| TC-UI-DASH-* | `DashboardPageProps` | 대시보드 컴포넌트 |
| TC-UI-FILE-* | `FileExplorerPageProps` | 파일 탐색기 컴포넌트 |
| TC-UI-FOLDER-* | `FolderSettingsPageProps` | 폴더 설정 컴포넌트 |
| TC-UI-LOG-* | `LogViewerPageProps` | 로그 뷰어 컴포넌트 |
| TC-UI-STAT-* | `StatisticsPageProps` | 통계 컴포넌트 |
| TC-UI-SET-* | `SettingsPageProps` | 설정 컴포넌트 |

> 📌 인터페이스 스펙 상세는 [10-SDD-개발방법론](../10-SDD-개발방법론.md) §5 모듈별 스펙 카탈로그를 참조한다.
