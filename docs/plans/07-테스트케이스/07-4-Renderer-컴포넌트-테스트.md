## 4. Renderer 컴포넌트 테스트

### 4.1 대시보드

> 상태 카드, 요약 정보, 최근 파일. 관련 기능: F-06

| TC ID | 테스트명 | 입력 | 기대 결과 | 우선순위 |
|-------|---------|------|----------|---------|
| TC-UI-DASH-001 | 동기화 상태 표시기 렌더링 | syncStatus='running' | 초록색 상태 뱃지 + "정상 동작 중" 텍스트 | P0 |
| TC-UI-DASH-002 | 오류 상태 표시 | syncStatus='error' | 빨간색 뱃지 + 오류 메시지 | P0 |
| TC-UI-DASH-003 | 오늘 동기화 요약 카드 | stats={total:15, success:13, failed:2} | "15건 / 성공 13 / 실패 2" 표시 | P0 |
| TC-UI-DASH-004 | 최근 동기화 파일 목록 (50건) | 최근 50건 파일 데이터 | 시간 역순 목록 렌더링, 파일명/상태/시간 표시 | P0 |
| TC-UI-DASH-005 | 진행 중 동기화 프로그레스 바 | downloadProgress={percentage:45} | 프로그레스 바 45% 표시 | P1 |
| TC-UI-DASH-006 | "전체 동기화" 버튼 클릭 | 버튼 클릭 | IPC invoke('sync:fullSync') 호출 | P0 |
| TC-UI-DASH-007 | "일시중지" 버튼 토글 | 버튼 클릭 | 상태에 따라 pause/resume IPC 호출 | P0 |
| TC-UI-DASH-008 | 외부웹하드 연결 상태 표시 | sessionValid=false | "연결 끊김" 표시 | P1 |
| TC-UI-DASH-009 | 빈 대시보드 (초기 상태) | 동기화 이력 없음 | 빈 상태 안내 메시지 표시 | P1 |
| TC-UI-DASH-010 | IPC 통신 실패 처리 | invoke 에러 | "상태 로딩 실패" + 새로고침 버튼 | P1 |
