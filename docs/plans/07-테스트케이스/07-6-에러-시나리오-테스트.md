## 6. 에러 시나리오 테스트

### 6.1 네트워크 에러

| TC ID | 테스트명 | 시나리오 | 기대 결과 | 우선순위 |
|-------|---------|---------|----------|---------|
| TC-E2E-ERR-001 | 네트워크 완전 끊김 | 모든 HTTP 요청 실패 Mock | 지수 백오프 재시도, 서킷 브레이커 OPEN, 사용자 알림 | P0 |
| TC-E2E-ERR-002 | 네트워크 복구 후 자동 재개 | 끊김 30초 후 복구 | 서킷 브레이커 HALF_OPEN -> CLOSED, 폴링 재개 | P0 |
| TC-E2E-ERR-003 | 간헐적 타임아웃 | 30% 확률로 타임아웃 Mock | 실패 건 재시도, 동시성 자동 감소, 정상 건은 처리 | P1 |

### 6.2 LGU+ API 에러

| TC ID | 테스트명 | 시나리오 | 기대 결과 | 우선순위 |
|-------|---------|---------|----------|---------|
| TC-E2E-ERR-004 | LGU+ 서버 500 에러 | 히스토리 API 500 반환 | 30초 간격 재시도, 3회 후 DLQ, 스냅샷 감지로 보완 | P0 |
| TC-E2E-ERR-005 | LGU+ API 응답 형식 변경 | 예상과 다른 JSON 구조 | 파싱 에러 로그 + 사용자 알림 | P1 |
| TC-E2E-ERR-006 | LGU+ 다운로드 URL 만료 | 다운로드 시도 시 403 | URL 재획득 후 재다운로드 | P1 |
| TC-E2E-ERR-007 | 다운로드 중 원본 파일 삭제 | 다운로드 중 404 | 건너뛰기 + 로그 기록 + 다음 파일 계속 | P1 |

### 6.3 자체웹하드 에러

| TC ID | 테스트명 | 시나리오 | 기대 결과 | 우선순위 |
|-------|---------|---------|----------|---------|
| TC-E2E-ERR-008 | 자체웹하드 서버 다운 | 업로드 API 연결 실패 | 30초 재시도, 5분 후 사용자 알림 | P0 |
| TC-E2E-ERR-009 | 업로드 체크섬 불일치 | 서버 응답 MD5 불일치 | 재다운로드 -> 재업로드 | P1 |
| TC-E2E-ERR-010 | Presigned URL 만료 | 업로드 시 URL 만료 | 새 URL 획득 후 재업로드 | P1 |

### 6.4 시스템 에러

| TC ID | 테스트명 | 시나리오 | 기대 결과 | 우선순위 |
|-------|---------|---------|----------|---------|
| TC-E2E-ERR-011 | 디스크 공간 부족 | 다운로드 중 디스크 풀 | 즉시 중단 + 사용자 알림 (재시도 없음) | P0 |
| TC-E2E-ERR-012 | SQLite DB 손상 | 손상된 DB 파일로 시작 | 백업 복원 시도 -> 실패 시 새 DB + 전체 동기화 | P1 |
| TC-E2E-ERR-013 | 동시 실행 방지 | 앱 2번째 인스턴스 실행 | 기존 인스턴스 활성화, 새 인스턴스 즉시 종료 | P0 |
| TC-E2E-ERR-014 | 파일 쓰기 권한 없음 | 읽기 전용 디렉토리에 다운로드 | FileSystemError -> 즉시 실패 + 사용자 알림 | P1 |
