### 비기능 요구사항 → 스펙 매핑

| 비기능 요구사항 | 스펙 정의 위치 | SDD Level |
|---|---|---|
| 성능 (동시 다운로드 5개) | `AppConfig.maxConcurrentDownloads` | Level 1 (타입) |
| 안정성 (서킷 브레이커) | `ICircuitBreaker` | Level 2 (인터페이스) |
| 보안 (자격증명 암호화) | `IConfigManager.encrypt/decrypt` | Level 2 (인터페이스) |
| 데이터 무결성 (MD5 체크) | `SyncFile.md5Hash` | Level 1 (타입) |
| 로깅 (30일 보존) | `AppConfig.logRetentionDays` | Level 1 (타입) |
| 에러 처리 체계 | `SyncAppError` (추상 클래스) | Level 3 (추상 클래스) |

> 스펙 추적 매트릭스의 상세 인터페이스 정의는 [10-SDD-개발방법론](../10-SDD-개발방법론.md) 모듈별 스펙 카탈로그를 참조한다.

## 부록 A: v1 → v2 주요 변경사항 요약

| 항목 | v1 | v2 |
|------|----|----|
| **UI** | CLI (터미널) | Electron + React (GUI) |
| **동기화 방향** | 양방향 설계 → 단방향 사용 | 단방향 전용 설계 |
| **아키텍처** | CLI → Core (2계층) | Renderer → Main → Core (3계층 IPC) |
| **배포** | Docker/PM2 | Windows 설치 프로그램 (.exe) |
| **운영** | 수동 CLI 명령 | 자동 시작 + 시스템 트레이 |
| **설정** | `.env` 파일 직접 편집 | GUI 설정 화면 |
| **알림** | CLI 콘솔 출력 | 인앱 + Windows 토스트 |
| **오류 대응** | 수동 CLI 명령 | 자동 복구 + GUI 재시도 버튼 |
| **소스 파일 수** | 70+ 파일 (레거시 잔존) | 최소화된 클린 구조 |

## 부록 B: 외부웹하드(LGU+) API 참고

> v1 코드베이스에서 추출한 비공식 API 정보. 역공학 기반이므로 변경될 수 있음.

### 기본 URL
- 메인: `https://only.webhard.co.kr`
- 파일 CDN: `https://whfile1.webhard.co.kr`

### 주요 엔드포인트

| 용도 | 엔드포인트 | 메서드 | 요청 형식 |
|------|-----------|--------|----------|
| 로그인 | `/login-process` | POST | `application/x-www-form-urlencoded` |
| 폴더 트리 조회 | `/wh` | POST | `{MESSAGE_TYPE: 'FOLDER', PROCESS_TYPE: 'TREE'}` |
| 파일 목록 조회 | `/wh` | POST | `{MESSAGE_TYPE: 'FOLDER', PROCESS_TYPE: 'LIST'}` |
| 업로드 이력 조회 | `/wh` | POST | `{MESSAGE_TYPE: 'USE_HISTORY', PROCESS_TYPE: 'LIST'}` |
| 다운로드 URL 획득 | `/downloads/{fileId}/server` | GET | query: `fileStatus=1` |
| 파일 다운로드 | `whfile1.webhard.co.kr/file/download` | GET | query 파라미터 |

### API 응답 공통 형식
```json
{
  "RESULT_CODE": "0000",
  "RESULT_MSG": "OK",
  "TOTAL": 150,
  "COUNT": 100,
  "ITEMS": [...]
}
```

- `RESULT_CODE`: `"0000"` = 성공, `"9999"` = 세션 만료
- 세션 만료 감지: HTTP 302 + Location에 'login' 포함, HTTP 401, HTML 응답(JSON 대신), RESULT_CODE 9999
