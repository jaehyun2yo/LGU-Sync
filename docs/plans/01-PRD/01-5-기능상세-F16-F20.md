### F-16. 자동 오류 복구

#### 설명

네트워크 단절, 세션 만료 등 일시적 오류를 자동으로 복구한다.

#### 사용자 스토리

- 사용자로서, 일시적 네트워크 오류로 인해 동기화가 영구 중단되지 않기를 원한다.
- 사용자로서, 세션 만료 시 자동 재로그인되기를 원한다.

#### 인수 기준

| ID | 기준 | 검증 방법 |
|----|------|-----------|
| F-16-AC01 | 네트워크 오류 시 지수 백오프로 자동 재시도한다 (1초 → 3초 → 10초 → 30초 → 최대 5분) | 네트워크 단절 시뮬레이션 후 재시도 패턴 확인 |
| F-16-AC02 | 세션 만료 감지 시 자동 재로그인 후 작업을 재개한다 | 세션 만료 후 폴링이 자동 재개되는지 확인 |
| F-16-AC03 | 실패한 개별 파일을 DLQ에 보관하고, 주기적으로 재시도한다 (1시간 간격) | DLQ에 쌓인 파일이 1시간 후 재시도되는지 확인 |
| F-16-AC04 | 5분 이상 stuck 상태인 이벤트를 자동 감지하여 재처리한다 | stuck 이벤트 자동 복구 확인 |
| F-16-AC05 | 자동 복구 시도 횟수와 결과를 로그에 기록한다 | 복구 시도 로그 확인 |

#### 엣지 케이스

- 장시간(1시간 이상) 네트워크 단절
- 자체웹하드 서버 장시간 다운
- 동시에 여러 종류의 오류 발생

#### 오류 시나리오

| 오류 | 처리 방안 |
|------|-----------|
| 자동 복구 최대 횟수 초과 | 사용자에게 수동 개입 필요 알림 |
| 복구 시도 중 다른 오류 발생 | 기존 오류 보존 + 새 오류 병행 처리 |

### F-17. 수동 재시도 기능

#### 설명

자동 복구에 실패한 작업을 사용자가 수동으로 재시도할 수 있다.

#### 사용자 스토리

- 사용자로서, 실패한 파일 목록을 확인하고 선택적으로 재시도하기를 원한다.
- 사용자로서, "전체 재시도" 버튼으로 모든 실패 건을 한 번에 재시도하기를 원한다.

#### 인수 기준

| ID | 기준 | 검증 방법 |
|----|------|-----------|
| F-17-AC01 | 실패 파일 목록을 별도 탭/패널로 표시한다 (파일명, 오류 내용, 실패 시간, 재시도 횟수) | 실패 파일이 목록에 표시되는지 확인 |
| F-17-AC02 | 개별 파일 재시도 버튼을 제공한다 | 재시도 후 성공 시 목록에서 제거되는지 확인 |
| F-17-AC03 | "전체 재시도" 버튼을 제공한다 | 모든 실패 파일이 재시도되는지 확인 |
| F-17-AC04 | 재시도 불가능한 오류(파일 삭제됨 등)는 구분하여 표시한다 | 재시도 불가 파일에 별도 표시가 되는지 확인 |
| F-17-AC05 | 실패 항목 삭제(무시) 기능을 제공한다 | 삭제 후 목록에서 제거되는지 확인 |

#### 엣지 케이스

- 수백 건의 실패 항목
- 재시도 중 추가 실패 발생
- 재시도 중 네트워크 단절

#### 오류 시나리오

| 오류 | 처리 방안 |
|------|-----------|
| 재시도 실패 | 재시도 횟수 증가 + 오류 메시지 갱신 |
| 원본 파일이 이미 삭제됨 | "파일 없음" 상태로 변경 + 재시도 비활성화 |

### F-18. SQLite 상태 관리

#### 설명

모든 동기화 상태, 이력, 설정을 SQLite 데이터베이스로 관리한다.

#### 사용자 스토리

- 사용자로서, 프로그램 재시작 후에도 모든 이력과 설정이 보존되기를 원한다.

#### 인수 기준

| ID | 기준 | 검증 방법 |
|----|------|-----------|
| F-18-AC01 | WAL(Write-Ahead Logging) 모드로 동작하여 읽기/쓰기 동시성을 보장한다 | 동시 읽기/쓰기 시 오류 없이 동작하는지 확인 |
| F-18-AC02 | 체크포인트 테이블: historyNo, 처리 시간, 상태 저장 | 체크포인트 CRUD 동작 확인 |
| F-18-AC03 | 동기화 이벤트 테이블: 이벤트 ID, 유형, 상태, 파일 정보, 타임스탬프 저장 | 이벤트 CRUD 동작 확인 |
| F-18-AC04 | 설정 테이블: 모든 사용자 설정 영속 저장 | 설정 변경 후 프로그램 재시작 시 유지 확인 |
| F-18-AC05 | 통계 테이블: 일별 동기화 건수, 용량, 성공률 집계 | 통계 조회 결과가 정확한지 확인 |
| F-18-AC06 | DLQ 테이블: 실패 이벤트 보관, 재시도 이력 | DLQ CRUD 및 재시도 이력 확인 |
| F-18-AC07 | 자동 마이그레이션: 스키마 버전 관리 및 업그레이드 | 스키마 변경 후 자동 마이그레이션 동작 확인 |
| F-18-AC08 | 자동 백업: 일 1회 DB 파일 백업 (최근 7개 보관) | 백업 파일 생성 및 보관 주기 확인 |

#### 엣지 케이스

- DB 파일 크기가 1GB를 초과하는 경우
- 디스크 공간 부족으로 쓰기 실패
- 동시에 여러 스레드에서 쓰기 시도

#### 오류 시나리오

| 오류 | 처리 방안 |
|------|-----------|
| DB 파일 손상 | 최근 백업에서 복원 + 사용자 알림 |
| 디스크 공간 부족 | 사용자 알림 + 오래된 이력 자동 정리 |
| 마이그레이션 실패 | 이전 버전 DB 백업 보존 + 오류 보고 |

### F-19. 자동 업데이트

#### 설명

프로그램의 새 버전을 자동으로 감지하고 업데이트한다.

#### 사용자 스토리

- 사용자로서, 새 버전이 나오면 자동으로 알림을 받고 업데이트하기를 원한다.

#### 인수 기준

| ID | 기준 | 검증 방법 |
|----|------|-----------|
| F-19-AC01 | GitHub Releases 또는 지정 서버에서 새 버전을 주기적으로 확인한다 | 새 버전 발행 후 프로그램에서 감지하는지 확인 |
| F-19-AC02 | 업데이트 가능 시 사용자에게 알림을 보낸다 (강제 아님) | 알림 표시 확인 |
| F-19-AC03 | "지금 업데이트" 버튼으로 다운로드 → 설치 → 재시작을 자동 수행한다 | 업데이트 완료 후 새 버전 확인 |
| F-19-AC04 | 업데이트 중 동기화를 일시중지하고, 업데이트 후 자동 재개한다 | 업데이트 전후 동기화 상태 확인 |

#### 엣지 케이스

- 업데이트 다운로드 중 네트워크 단절
- 업데이트 파일 무결성 검증 실패
- 업데이트 후 DB 스키마 마이그레이션 필요

#### 오류 시나리오

| 오류 | 처리 방안 |
|------|-----------|
| 업데이트 서버 접속 불가 | 무시하고 현재 버전으로 계속 운영 |
| 설치 실패 | 이전 버전으로 롤백 + 오류 보고 |

### F-20. 내부웹하드 API 어댑터

#### 설명

자체웹하드 API와의 통신을 인터페이스로 추상화하여, API가 변경되거나 다른 백엔드로 교체되어도 핵심 로직에 영향이 없도록 한다.

#### 사용자 스토리

- 개발자로서, 자체웹하드 API가 변경되어도 어댑터 구현체만 교체하면 되기를 원한다.

#### 인수 기준

| ID | 기준 | 검증 방법 |
|----|------|-----------|
| F-20-AC01 | `IInternalWebhardApi` 인터페이스가 정의되어 있다 (폴더 조회, 폴더 생성, 파일 업로드, 파일 존재 확인, 파일 삭제) | 인터페이스 정의 코드 리뷰 |
| F-20-AC02 | 현재 yjlaser_website API 용 구현체가 존재한다 (`YjlaserWebhardAdapter`) | 구현체 코드 리뷰 + 통합 테스트 |
| F-20-AC03 | Mock 구현체가 존재하여 단위 테스트에 사용할 수 있다 | Mock을 사용한 테스트 실행 확인 |
| F-20-AC04 | 설정에서 API 어댑터를 선택할 수 있다 (현재는 yjlaser만 제공) | 설정 UI에서 어댑터 선택 옵션 확인 |
| F-20-AC05 | 어댑터 교체 시 동기화 핵심 로직(`SyncEngine`)에 코드 변경이 필요 없다 | 다른 어댑터로 교체 후 테스트 통과 확인 |

#### 엣지 케이스

- API 응답 형식이 예상과 다른 경우
- 어댑터 초기화 실패

#### 오류 시나리오

| 오류 | 처리 방안 |
|------|-----------|
| API 응답 파싱 실패 | 상세 오류 로그 + 원본 응답 보존 |
| 인터페이스 미구현 메서드 호출 | TypeScript 컴파일 타임에 차단 |
