# 022. 다운로드 파이프라인 서킷 브레이커 + 디스크 검증 수정

- **날짜:** 2026-03-10
- **브랜치:** master

## 변경 요약
다운로드 테스트에서 성공 3137건 표시되지만 실제 파일 0개, 폴더 478개만 생성되는 버그 수정. 서킷 브레이커 과도 카운팅, DB 상태 재활용 시 디스크 검증 누락, fetch redirect 미처리가 복합적으로 작용한 문제.

## 변경 파일
- `src/core/retry-manager.ts` — retry 루프 내 recordFailure() 중복 호출 제거, 최종 실패 시 1회만 기록. failureThreshold 5→15 상향
- `src/core/lguplus-client.ts` — downloadFile fetch에 redirect:'manual' 추가 + 3xx 시 AuthSessionExpiredError throw, getDownloadUrlInfo catch에 logger.warn 추가, callWhApi 빈 응답을 ApiResponseParseError로 처리
- `src/main/ipc-router.ts` — DB 중복 체크에 fs.existsSync 디스크 검증 추가, 파일 미존재 시 status를 'detected'로 리셋하여 재다운로드

## 주요 결정사항
- 서킷 브레이커 threshold를 5→15로 상향한 이유: 3000+건 다운로드에서 5건 실패로 전체 중단은 과도. 15건으로 해도 실제 서버 장애 시 충분히 빠르게 차단됨
- retry 루프 내 매 시도마다 recordFailure 호출 대신 최종 실패 시에만 1회 기록: 1파일 실패 = 1 failure가 직관적이고 threshold 계산이 예측 가능
- 빈 응답을 에러로 처리: callWhApi는 DATA 요청 전용이므로 빈 응답은 비정상 (DATA-1 이슈)
