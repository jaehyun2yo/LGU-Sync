# 007. 공유 폴더스캔 캐시 + 실시간 감지 테스트 탭

- **날짜:** 2026-03-09
- **브랜치:** master

## 변경 요약
FolderTreeCache에 30분 TTL 스캔 결과 캐시를 추가하여 TestPage 탭 간 폴더 스캔 결과를 공유. 실시간 감지 테스트 탭(4번째)을 추가하여 FileDetector 기반 폴링+자동 다운로드/업로드/알림을 TestPage에서 실행 가능하게 구현.

## 변경 파일
- `src/core/folder-tree-cache.ts` — `getScanResult`/`setScanResult`/`invalidateScanResult` 메서드 추가 (30분 TTL)
- `src/core/container.ts` — `CoreServices`에 `folderCache` 필드 노출, `scanResultTtlMs` 옵션 전달
- `src/shared/ipc-types.ts` — `test:scan-folders` 응답 형식 변경 (`{folders, cachedAt}`), `forceRefresh` 요청 지원, `RealtimeTestStartRequest`/`RealtimeTestEvent` 타입, `test:realtime-start`/`stop` 채널, `test:realtime-event` 이벤트 추가
- `src/main/ipc-router.ts` — `test:scan-folders`에 캐시 로직 적용, `test:realtime-start`/`stop` 핸들러 구현 (폴링+감지+다운로드/업로드+OS알림)
- `src/renderer/pages/TestPage.tsx` — `cachedAt` 상태+새로고침 버튼+`formatTimeAgo`, 탭 전환 시 자동 selecting, 실시간 감지 탭 UI (옵션 체크박스, 시작/중지 토글, 이벤트 로그)

## 주요 결정사항
- 별도 `ScanResultCache` 클래스를 만들지 않고 기존 `FolderTreeCache`를 확장하여 DI 구조 변경 최소화
- 실시간 감지는 `setInterval` 기반 폴링으로 구현 (기존 `FileDetector.forceCheck()` 재사용)
- `test:realtime-stop`에서 `BrowserWindow.getAllWindows()`로 윈도우 참조 (registerIpcHandlers에 win 파라미터 없으므로)
