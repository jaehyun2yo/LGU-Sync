# 010. 스냅샷 감지 전략 추가

- **날짜:** 2026-03-09

## 변경 요약
FileDetector에 snapshot 기반 감지 전략을 추가하여 거래처(게스트) 업로드를 포함한 모든 외부웹하드 변동을 감지할 수 있게 함. 기존 polling(history 기반) 전략은 선택 가능하게 유지하되, 기본 전략을 snapshot으로 전환.

## 변경 파일
- `src/core/snapshot-diff.ts` — diffSnapshot 폴더 파일 목록 비교 함수 (신규)
- `src/core/file-detector.ts` — strategy 옵션 + pollBySnapshot 메서드 추가
- `src/shared/ipc-types.ts` — RealtimeTestStartRequest에 strategy 필드 추가
- `src/main/ipc-router.ts` — test:realtime-start 핸들러에서 strategy 전달
- `src/renderer/pages/TestPage.tsx` — strategy 라디오 버튼 UI 추가
- `src/core/container.ts` — DI 컨테이너 기본 전략 snapshot 전환
- `tests/core/file-detector-limitations.test.ts` — 기존 polling 한계 증명 테스트 11개 (신규)
- `tests/core/snapshot-detector.test.ts` — diffSnapshot 단위 테스트 8개 (신규)
- `tests/core/file-detector-snapshot.test.ts` — snapshot 전략 통합 테스트 8개 (신규)

## 주요 결정사항
- `getUploadHistory()`는 로그인 유저 이력만 반환하므로 거래처 업로드 감지 불가 → 한계 테스트로 증명
- `getFileList(folderId)`로 폴더 내용을 직접 비교하는 snapshot 방식이 유일한 해결책
- 기존 polling 전략은 제거하지 않고 선택 가능하게 유지 (하이브리드 구조)
- snapshot을 기본 전략으로 설정하되, UI에서 polling으로 전환 가능
