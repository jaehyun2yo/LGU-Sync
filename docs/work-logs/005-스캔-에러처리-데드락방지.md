## 변경 요약
LGU+ 웹하드 API 호출 시 네트워크 에러로 인한 unhandled promise rejection과 getAllFilesDeep 워커 풀 데드락 문제를 수정.

## 변경 파일
- `src/core/lguplus-client.ts` — callWhApi에 fetch/JSON.parse 에러 변환 추가, getAllFiles 배치 실패 graceful degradation, getAllFilesDeep processNext에 try-catch-finally로 데드락 방지
- `src/main/ipc-router.ts` — buildSubFolderTree Promise.allSettled 전환, collectAllFolderPaths/test:download-only/test:full-sync에 스캔 try-catch 추가
- `tests/core/lguplus-client-scan.test.ts` — 네트워크 에러 변환, 배치 실패 부분 결과, 데드락 방지 테스트 7건 추가

## 주요 결정사항
- **RetryManager 미적용**: 스캔은 벌크 읽기이므로 개별 폴더 실패는 skip하고 다음 폴더 처리. 사용자가 재스캔으로 전체 재시도.
- **getAllFiles에서 break**: 배치 실패 시 수집된 파일만 반환 (80% > 0%).
- **finally 블록**: activeWorkers--와 checkDone()을 finally로 이동하여 에러 시에도 워커 카운터 정리 보장.
