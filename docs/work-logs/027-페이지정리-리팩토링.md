## Key Decisions

1. **ESLint flat config 마이그레이션**: ESLint 10이 .eslintrc.cjs를 지원하지 않아 eslint.config.mjs로 마이그레이션. 기존 규칙 그대로 유지.

2. **기존 lint 에러 분리**: 수정되지 않은 파일(config-manager.ts, container.ts, webhard-uploader, index.ts, tray-manager.ts, window-manager.ts)의 기존 lint 에러는 이번 리팩토링 범위 밖으로 남겨둠. (기존: 45개 에러, 현재: 11개 에러)

3. **syncFile 변경 반영**: sync-engine.ts의 syncFile이 download only + completed (upload 없음) 구조로 변경됨. pipeline-integration.test.ts의 upload 관련 테스트를 새 동작에 맞게 수정.

4. **discoverAncestorPrefix 추가로 인한 테스트 수정**: folder-discovery.ts에 discoverAncestorPrefix 메서드가 추가되어 getSubFolders를 한 번 더 호출함. 테스트의 mockResolvedValueOnce 순서를 보정.
