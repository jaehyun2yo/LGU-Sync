## 주요 결정사항

### 게스트 폴더 API 제약

API 역분석 결과, 게스트 폴더에서는 **FOLDER/MAKE(생성)만** 정상 동작:
- DELETE: 빈 200 응답 반환 (삭제 안 됨, callWhApi가 `RESULT_CODE: "0000"`으로 변환해 성공처럼 보임)
- RENAME/MOVE: `-9999` 에러 반환
- 파일 업로드/삭제/이름변경/이동: 미테스트 (게스트 폴더 권한 부족)

따라서 계획의 8가지 CRUD 테스트를 **폴더 생성 감지**로 축소했다.

### FOLDER/MAKE API 파라미터

`FOLDER_NAME`이 아닌 `NAME` 파라미터를 사용해야 한다. `FOLDER_NAME`은 null 이름 폴더가 생성됨. `scripts/final-api-probe.ts`로 발견.

### InMemoryStateManager

better-sqlite3 네이티브 모듈 버전 불일치(NODE_MODULE_VERSION 127)를 회피하기 위해 인메모리 구현 사용. FileDetector polling 전략은 checkpoint read/write만 필요.

### 단일 공유 컨텍스트

파일 레벨 `beforeAll`/`afterAll`로 로그인 1회만 수행. describe 블록별 별도 로그인은 동시 세션 충돌 발생.
