## 주요 결정사항

1. **DetectionService 도입**: SyncEngine 위에 감지 생명주기를 관리하는 상위 서비스 레이어. 세션 추적, 다운타임 복구, DLQ 재시도를 담당
2. **다운타임 복구 방식**: `stopped_at` NULL 여부로 비정상 종료 감지 → `last_history_no` 기반 checkpoint 롤백 → forceCheck()로 누락 파일 재스캔 (최대 7일)
3. **IPC 채널 네이밍**: `test:realtime-*` → `detection:*` 로 전환. 프로덕션 기능에 적합한 네임스페이스
4. **MAX_POLL_PAGES 증가**: 10→50 (최대 1000건 스캔). 대량 업로드 시 감지 누락 방지
5. **DLQ 자동 재시도**: 5분 주기로 실패 파일 자동 재시도
