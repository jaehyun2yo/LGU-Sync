# 006. 비DB 폴더 자동등록 — 업로드 에러 수정

- **날짜:** 2026-03-09
- **브랜치:** master

## 변경 요약
TestPage에서 비DB 폴더(lguplus: 접두사) 선택 시 다운로드→업로드 흐름이 실패하던 문제 수정. 비DB 폴더를 자동으로 DB에 등록하여 state 추적 경로로 통합.

## 변경 파일
- `src/main/ipc-router.ts` — `test:download-only` 핸들러: 비DB 폴더 자동등록 로직 추가(780-795행), else 분기(직접 다운로드) 제거하여 모든 폴더가 `engine.downloadOnly` + state 추적 경로로 처리
- `src/main/ipc-router.ts` — `test:full-sync` 핸들러: `lguplus:` 접두사 폴더 ID 해석 로직 추가(999-1031행), 비DB 폴더 자동등록 포함
- `src/main/ipc-router.ts` — `SyncFolderRow` 타입 import 추가

## 주요 결정사항
- **자동등록 위치를 파일 루프 밖으로 이동:** 폴더당 한 번만 `saveFolder` 호출하도록 최적화. `getFolderByLguplusId`로 중복 확인 후 등록하여 UNIQUE 제약 위반 방지.
- **else 분기 완전 제거:** 자동등록 후 모든 폴더가 DB 폴더로 전환되므로, 기존의 직접 다운로드(state 미추적) 경로 불필요. 코드 중복 제거 및 업로드 호환성 확보.
- **full-sync도 동일 패턴 적용:** download 핸들러와 동일하게 `lguplus:` 접두사 파싱 + 자동등록 패턴 사용.
