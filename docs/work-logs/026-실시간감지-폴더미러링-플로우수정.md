# 026. 실시간 감지 로직 플로우 수정 — 폴더 미러링 + 경로 버그 수정

- **Date:** 2026-03-12
- **Branch:** master

## Summary
실시간 감지 시작 시 외부웹하드 전체 폴더구조를 로컬 다운로드 폴더에 1:1 미러링하는 플로우를 보강하고, SyncEngine의 4가지 버그를 수정.

## 변경 파일

### `src/core/sync-engine.ts`
- **문제 1 (Critical)**: `scanFolder()`의 `file_path` 구성 시 `lguplus_folder_name`(이름만) → `lguplus_folder_path`(전체 경로) 사용으로 변경. null이면 `lguplus_folder_name` 폴백.
- **문제 2 (Medium)**: `getRootFolders()` private 메서드 추가 — 다른 폴더의 하위가 아닌 최상위 폴더만 반환하여 fullSync 시 중복 스캔 방지.
- **문제 3 (Low)**: `createDownloadDirs()` private 메서드 추가 — fullSync 시작 시 등록된 모든 폴더의 로컬 디렉토리 사전 생성. `filterPathSegments()`로 GUEST 세그먼트 제거하여 `downloadOnly()`와 경로 일치.
- **문제 4 (Medium)**: `handleFolderCreate()`를 async로 변경 — FC 이벤트 시 미등록 폴더를 `saveFolder()`로 DB 등록 + 로컬 다운로드 디렉토리 생성. `cleanFolderPath()`로 경로 정규화.
- **안전망**: `downloadOnly()`에 `mkdir(dirname(destPath), { recursive: true })` 추가 — startup mirroring 실패 시에도 다운로드 보장.
- `import` 추가: `mkdir` (fs/promises), `dirname` (path), `cleanFolderPath` (path-utils)

### `src/core/detection-service.ts`
- `createDownloadFolders()`에 `filterPathSegments()` 적용으로 GUEST 세그먼트 제거, `downloadOnly()`와 경로 일관성 확보
- 개별 폴더 생성 실패 시 `failed` 카운터 로깅 추가

### `tests/core/sync-engine.test.ts`
- `scanFolder file_path 경로 수정` 스위트: lguplus_folder_path 사용 + null 폴백 (2개)
- `getRootFolders() 중복 스캔 방지` 스위트: 중첩 폴더 필터링, 독립 폴더 모두 스캔, folderIds 지정 (3개)
- `FC operCode 폴더 등록` 스위트: 미등록 폴더 DB 저장, 이미 등록된 폴더 무시, FC→UP 연속 시나리오 (3개)
- 기존 테스트의 폴더 mock에 `lguplus_folder_path` 필드 추가

## 주요 결정사항
- `getRootFolders()`는 `lguplus_folder_path.startsWith(other + '/')` 패턴으로 중첩 판별 — 단순하지만 정확. O(n²)이나 폴더 수가 적어 문제 없음.
- `downloadOnly()`에 `mkdir` 안전망을 추가한 이유: discoverFolders() 실패(네트워크 등) 시에도 개별 파일 다운로드는 성공해야 함. 이중 보호 전략.
- `handleFolderCreate()`의 fire-and-forget 패턴: FC 이벤트는 폴더 등록이 다음 파일 감지 전에 완료되어야 하지만, 비동기 처리 중 에러가 전체 감지를 중단하면 안 됨.
