# 018. 통합 알림 시스템 구현

- **날짜:** 2026-03-10

## 변경 요약
이벤트별(감지/완료/실패/세션만료) × 채널별(소리/토스트/인앱) 매트릭스 기반 통합 알림 시스템 구현.
Electron 네이티브 Notification을 제거하고 Renderer-side에서 Web Audio API 알림음 + 커스텀 토스트 + 인앱 알림으로 통합.

## 변경 파일
- `src/core/types/config.types.ts` — NotificationEventType, EventNotificationRule, SoundPresetId, NotificationConfig 타입 추가
- `src/core/config-manager.ts` — DEFAULT_CONFIG.notification 확장, Zod 스키마 업데이트
- `src/shared/ipc-types.ts` — AppSettings.notification을 NotificationConfig로 교체, re-export 추가
- `src/renderer/lib/notification-sound.ts` — (신규) Web Audio API OscillatorNode 기반 5개 프리셋 알림음 합성
- `src/renderer/stores/toast-store.ts` — (신규) 토스트 상태 관리 (entering→visible→exiting 상태 머신)
- `src/renderer/components/ToastContainer.tsx` — (신규) 토스트 UI (타입별 아이콘/색상, 프로그레스바, CSS transition)
- `src/renderer/components/Layout.tsx` — ToastContainer 추가
- `src/renderer/stores/notification-store.ts` — addNotification 로컬 추가 액션
- `src/renderer/lib/notification-manager.ts` — (신규) useNotificationManager 훅 (IPC 이벤트 → 설정 규칙 → 채널 분기)
- `src/renderer/App.tsx` — useNotificationManager 훅 호출
- `src/renderer/pages/SettingsPage.tsx` — NotificationTab 전면 재설계 (마스터 토글, 알림음 선택/미리듣기/볼륨, 이벤트별 규칙 매트릭스)
- `src/renderer/stores/settings-store.ts` — notification deep merge 적용
- `src/main/ipc-router.ts` — Electron 네이티브 Notification 코드 제거
- `tests/renderer/stores/toast-store.test.ts` — (신규) 5개 테스트
- `tests/renderer/lib/notification-sound.test.ts` — (신규) 6개 테스트

## 주요 결정사항
- **Renderer-side 오케스트레이션**: Main 프로세스 변경 최소화를 위해 알림 로직을 Renderer에서 처리. IPC 이벤트를 useNotificationManager가 구독하여 설정에 따라 분기.
- **Web Audio API**: 외부 사운드 파일 없이 OscillatorNode로 5가지 프리셋을 합성. 앱 번들 크기 영향 없음.
- **CSS transition 토스트**: 라이브러리 의존성 없이 순수 CSS transition으로 슬라이드 애니메이션 구현.
- **Deep merge**: notification 설정이 중첩 객체로 변경되어 settings-store의 updateSettings에 notification 전용 deep merge 로직 추가.
