## 주요 결정사항
- **공통 useSort 훅:** 제네릭 `<T, K>` 설계로 모든 테이블에 재사용. comparator를 외부에서 주입하여 정렬 로직 커스터마이징 가능.
- **트리 구성 방식:** `getAllFilesDeep()` 대신 `getAllFiles()` (1단계 파일) + `buildSubFolderTree()` (재귀 서브폴더)로 분리. 트리 구조와 파일 수를 정확히 표시하기 위함.
- **downloadOnly 경로 보존:** `file_path`에서 세그먼트를 파싱하여 `destPath`에 반영. `lguplus-client.ts` 내부의 `mkdir(dirname, { recursive: true })`가 서브폴더를 자동 생성하므로 추가 처리 불필요.
- **uploadOnly 경로 보존:** `file_path` 세그먼트를 `ensureFolderPath()`에 전달. 기존 캐시(`self_webhard_path`)는 서브폴더가 없는 경우에만 폴백으로 사용.
