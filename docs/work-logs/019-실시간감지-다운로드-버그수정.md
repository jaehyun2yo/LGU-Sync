# 019. 실시간 감지 → 다운로드 버그 수정

- **날짜:** 2026-03-10
- **브랜치:** fix/realtime-detection-download-bugs

## 변경 요약

실시간 파일 감지 후 다운로드 실패, 확장자 중복, 감지 중단 3가지 버그를 수정.
서킷 브레이커 상태를 UI에 노출하여 수동 리셋 가능하도록 개선.
**근본 원인 수정**: 폴링 감지 시 `UploadHistoryItem.itemSrcNo`(LGU+ 파일 ID)가 `DetectedFile`에 전달되지 않아 `downloadOnly()`가 `historyNo`를 파일 ID로 사용하던 버그 수정.

## 주요 결정사항

- **B1 (확장자 중복)**: `itemSrcName`이 이미 `.확장자`로 끝나는지 대소문자 무시 비교로 판단. 빈 확장자(`''`)는 dot 없이 이름만 사용.
- **B2 (다운로드 실패)**: `{success:false}` 반환 → `throw`로 변경하여 RetryManager가 실패로 인식하고 서킷 브레이커가 정상 작동하도록 함. 기존 에러 계층 `FileDownloadUrlFetchError`(retryable=true) 활용.
- **B4 (itemSrcNo 유실 — 근본 원인)**: `UploadHistoryItem.itemSrcNo`(실제 파일 ID)가 `DetectedFile`에 전달되지 않아 `downloadOnly()`가 `historyNo`(이력 순번)를 `getDownloadUrlInfo()`에 전달. 3개 레이어(DetectedFile → startFileSync → downloadOnly) 수정으로 `itemSrcNo` 전달 경로 확보. `fullSync`는 `scanFolder()`에서 `lguplus_file_id: String(file.itemId)`로 정상 설정되어 영향 없었음.
- **B3 (서킷 UI)**: `getAllCircuitStates()` 추가로 UI에서 OPEN 서킷 확인 가능. `sync:reset-circuit` IPC로 수동 리셋 지원. 성공-only 서킷은 맵에 등록되지 않으므로 반환 안 함 (의도된 동작).
