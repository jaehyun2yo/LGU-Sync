# 003. 로컬 파일 존재 시 다운로드 스킵 기능

- **날짜:** 2026-03-08
- **브랜치:** master

## 변경 요약
로컬 다운로드 폴더에 동일 파일(파일명+크기)이 이미 존재하면 다운로드를 건너뛰어 동기화 속도를 개선한다. `SyncResult`에 `skipped` 필드를 추가하여 스킵 여부를 추적.

## 변경 파일
- `src/core/types/sync-engine.types.ts` — `SyncResult.skipped?: boolean` 추가
- `src/core/sync-engine.ts` — `import { stat }`, `isLocalFileValid()` private 메서드 추가, `downloadOnly()` 상단에 스킵 로직 삽입
- `tests/core/sync-engine.test.ts` — 다운로드 스킵 테스트 4개 추가 (파일 존재+크기 일치, 파일 미존재, 크기 불일치, file_size=0)

## 주요 결정사항
- `fs.stat()`으로 파일 존재+크기 비교. 해시 비교는 성능 대비 이점이 적어 제외.
- `file_size`가 0인 경우 크기 비교 없이 파일 존재만으로 스킵 (LGU+ API에서 크기 정보가 없는 경우 대응).
- `download_path`가 이미 있으면 해당 경로를, 없으면 `tempPath/file_name` 경로를 확인.
- ESM 환경에서 `vi.spyOn`이 `node:fs/promises`에 동작하지 않아 `vi.mock` + `vi.importMock` 패턴 사용.
