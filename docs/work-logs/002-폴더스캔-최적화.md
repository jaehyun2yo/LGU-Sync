# 002. 폴더 스캔 최적화

- **날짜:** 2026-03-08
- **브랜치:** feature/folder-scan-optimization

## 변경 요약
fullSync 시 폴더 스캔 속도를 대폭 개선. Worker pool BFS, 페이지네이션 병렬 fetch, 폴더 구조 캐싱, fullSync/FolderDiscovery 병렬 처리를 도입하여 예상 3~10배 성능 향상.

## 변경 파일
- `src/core/lguplus-client.ts` — getAllFilesDeep을 worker pool BFS로 교체, getAllFiles 페이지네이션 병렬 fetch, getSubFolders에 FolderTreeCache 적용, 생성자에 캐시 옵셔널 파라미터 추가
- `src/core/folder-tree-cache.ts` — (신규) 폴더 구조 메모리 캐싱 클래스 (TTL 기반, getSubFolders/fileCount 캐시)
- `src/core/container.ts` — FolderTreeCache 인스턴스 생성 및 LGUplusClient DI 주입
- `src/core/sync-engine.ts` — fullSync의 for-of 순차 루프를 Promise.allSettled 배치 병렬 처리(concurrency=3)로 교체, scanFolder 프라이빗 메서드 추출
- `src/core/folder-discovery.ts` — 새 폴더 ensureFolderPath를 병렬 배치 처리(concurrency=3), processNewFolder 프라이빗 메서드 추출
- `tests/core/lguplus-client-scan.test.ts` — (신규) worker pool BFS 3개, 페이지네이션 2개, 캐시 통합 2개 테스트
- `tests/core/folder-tree-cache.test.ts` — (신규) FolderTreeCache 단위 테스트 7개
- `tests/core/folder-discovery.test.ts` — (신규) FolderDiscovery 병렬 처리 테스트 2개
- `tests/core/sync-engine.test.ts` — fullSync 병렬 스캔 테스트 2개 추가

## 주요 결정사항
- **Worker pool vs level-by-level BFS:** 기존 레벨 단위 BFS는 한 레벨이 끝나야 다음 레벨이 시작되어 병렬성이 제한됨. Worker pool 패턴으로 폴더 발견 즉시 큐에 넣고 빈 워커가 처리하도록 변경하여 깊은 트리에서 대기 시간 제거.
- **캐시를 별도 클래스로 분리:** LGUplusClient에 직접 캐시 로직을 넣지 않고 FolderTreeCache를 독립 클래스로 만들어 테스트 용이성 확보. 옵셔널 파라미터로 주입하여 기존 코드와 하위 호환성 유지.
- **Promise.allSettled 사용:** Promise.all 대신 allSettled를 사용하여 한 폴더 실패가 다른 폴더 처리에 영향을 주지 않도록 격리.
- **Concurrency 제한(3):** API 서버 부하를 고려하여 무제한 병렬이 아닌 배치 단위(3개씩) 병렬 처리.
