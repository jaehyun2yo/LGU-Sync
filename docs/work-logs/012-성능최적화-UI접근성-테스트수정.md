# 012. 성능 최적화, UI 접근성 개선, 테스트 수정

- **날짜:** 2026-03-09
- **브랜치:** master

## 변경 요약
4인 에이전트 팀으로 프로젝트 전체 코드를 병렬 리뷰하고, 성능(5건), UI/UX(6건), 테스트(1건) 문제를 수정했다. 아키텍처는 수정 불필요로 판단.

## 변경 파일

### 성능 최적화
- `src/core/lguplus-client.ts` — downloadFile을 createWriteStream 스트리밍으로 전환, batchDownload를 worker pool 병렬 처리, getAllFilesDeep busy-wait를 이벤트 기반 알림으로 전환
- `src/core/webhard-uploader/yjlaser-uploader.ts` — uploadFile을 createReadStream 스트리밍으로 전환, uploadFileBatch를 worker pool 병렬 처리

### UI/UX 접근성
- `src/renderer/components/Header.tsx` — pageTitles에 migration, test 페이지 추가
- `src/renderer/components/ConfirmDialog.tsx` — role="dialog", aria-modal, aria-labelledby, 포커스 트래핑, Escape 키 닫기 추가
- `src/renderer/components/NotificationCenter.tsx` — role="dialog", aria-label, Escape 키 닫기 추가
- `src/renderer/components/Sidebar.tsx` — 접기/펼치기 버튼에 aria-label 추가
- `src/renderer/App.tsx` — 테마 초기화 중복 useEffect 제거
- `src/renderer/stores/ui-store.ts` — getInitialTheme에서 초기 DOM 클래스 설정
- `src/renderer/lib/utils.ts` — formatElapsedTime 함수 추가
- `src/renderer/pages/MigrationPage.tsx` — 중복 formatDuration 제거, lib/utils 사용
- `src/renderer/pages/TestPage.tsx` — 중복 formatDuration, formatBytes, formatSize 제거, lib/utils 사용

### 테스트 수정
- `tests/core/config-manager.test.ts` — password 기대값을 DEFAULT_CONFIG 참조로 수정
- `tests/core/lguplus-client.test.ts` — createWriteStream mock 추가
- `tests/core/yjlaser-uploader.test.ts` — createReadStream mock 추가
- `tests/core/yjlaser-uploader-batch-record.test.ts` — 동일 mock 업데이트

## 주요 결정사항
- 대용량 파일 다운로드/업로드 시 전체 파일을 메모리에 버퍼링하던 방식을 스트리밍으로 전환하여 힙 초과 크래시 방지
- batchDownload/uploadFileBatch의 순차 처리를 worker pool 패턴(concurrency 3)으로 병렬화
- 아키텍처 리뷰 결과 DI 컨테이너, 인터페이스 설계, IPC 패턴 등 핵심 설계가 견고하여 수정 불필요
