# 004. 폴더 구조 보존 다운로드 + 최신 데이터 검증

- **날짜:** 2026-03-08
- **브랜치:** feature/structured-download-path (master에 머지 완료)

## 변경 요약
다운로드 경로에 외부웹하드 폴더 구조를 보존하고, `lguplus_updated_at` 컬럼을 추가하여 기존 파일의 최신 여부를 검증한 뒤 스킵 또는 재다운로드하도록 개선.

## 변경 파일
- `src/core/db/schema.ts` — `sync_files` 테이블에 `lguplus_updated_at` 컬럼 추가 + `MIGRATIONS` 배열 추가
- `src/core/db/types.ts` — `SyncFileRow`, `SyncFileInsert`, `SyncFileInsertSchema`에 `lguplus_updated_at` 필드 추가
- `src/core/state-manager.ts` — `saveFile()` INSERT에 `lguplus_updated_at` 추가, `updateFileStatus()` allowedFields에 추가, `initialize()`에 마이그레이션 실행 로직 추가
- `src/core/sync-engine.ts` — `downloadOnly()`에서 `file_path` 기반 경로 구성, `fullSync()`에서 `updatedAt` 비교 로직 추가
- `tests/core/sync-engine.test.ts` — 폴더 구조 보존 테스트 1개, updatedAt 비교 테스트 2개 추가

## 주요 결정사항
- 다운로드 경로: `${tempPath}/${file.file_name}` → `${tempPath}${file.file_path}` — `file_path`가 이미 `/`로 시작하므로 자연스럽게 연결
- `lguplus-client.ts`의 `downloadFile()`이 이미 `mkdir(dirname(destPath), { recursive: true })`를 호출하므로 중간 폴더 자동 생성 별도 처리 불필요
- `history_no` UNIQUE 인덱스로 인해 업데이트된 파일은 새 INSERT 대신 기존 레코드의 status를 `detected`로 리셋 후 재동기화
- 마이그레이션은 `try/catch`로 감싸서 이미 적용된 경우 무시 (기존 DB 호환)
