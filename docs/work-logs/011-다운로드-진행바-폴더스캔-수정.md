# 011. 다운로드 진행바 + 폴더 스캔 정확도 수정

- **날짜:** 2026-03-09
- **브랜치:** fix/download-progress-and-folder-scan

## 변경 요약
다운로드 진행 시 진행바가 0%→100%로 점프하던 문제를 스트리밍 다운로드로 전환하여 실시간 퍼센티지 표시되도록 수정. 폴더 스캔 시 파일 개수/용량이 부정확하던 문제를 getAllFiles + 폴더 필터링으로 수정.

## 변경 파일
- `src/core/lguplus-client.ts` — downloadFile()을 ReadableStream 기반 스트리밍으로 변경, 200ms 스로틀 onProgress 콜백 추가. getAllFiles() 배치 실패 시 break→개별 페이지 재시도로 변경.
- `src/core/sync-engine.ts` — downloadOnly()에서 onProgress 콜백을 EventBus `sync:progress`로 연결
- `src/renderer/stores/sync-store.ts` — fullSyncProgress를 바이트 기반 계산으로 변경 (fallback: 파일 수 기반)
- `src/main/ipc-router.ts` — migration:scan에서 getFileList()→getAllFiles() 변경, isFolder 필터링, totalSize 실제 계산
- `tests/core/lguplus-client-download.test.ts` — 스트리밍 다운로드 진행 콜백 테스트 3개 (신규)
- `tests/core/lguplus-client-pagination.test.ts` — 페이지네이션 에러 복원력 테스트 3개 (신규)
- `tests/core/sync-engine.test.ts` — downloadFile 호출 시 onProgress 인자 기대값 추가

## 주요 결정사항
- 스트리밍 다운로드 시 `res.body` null인 경우 기존 arrayBuffer fallback 유지 (호환성)
- 진행 이벤트 스로틀을 200ms로 설정 (너무 잦은 UI 업데이트 방지)
- getAllFiles 배치 실패 시 전체 중단(break) 대신, 실패 배치의 개별 페이지를 순차 재시도하여 최대한 많은 데이터 수집
- migration:scan에서 getAllFiles를 사용하므로 API 호출이 증가하지만, 스캔은 비빈번 작업이므로 정확도 우선
