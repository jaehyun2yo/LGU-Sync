## 1. 프로그램 개요

### 목적

LGU+ 외부웹하드에 업로드된 파일을 자동으로 감지하여, 유진레이저 자체웹하드로 동기화하는 데스크톱 프로그램입니다.

### 동기화 방향

```
LGU+ 외부웹하드 (only.webhard.co.kr)
        │
        │  다운로드
        ▼
   로컬 임시 폴더 (./downloads/)
        │
        │  업로드 (R2 Presigned URL)
        ▼
유진레이저 자체웹하드 (yjlaser.net)
```

### 두 가지 동기화 방식

| 방식 | 설명 | 용도 |
|------|------|------|
| **실시간 폴링** | 5초마다 LGU+ 업로드 히스토리를 확인하여 신규 파일을 자동 동기화 | 일상적인 자동 동기화 |
| **전체 동기화** | 활성화된 모든 폴더의 전체 파일 목록을 스캔하여 누락된 파일을 일괄 동기화 | 초기 설정 또는 누락 파일 보정 |

## 2. 화면 구성

### 전체 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│ ┌────────────┐ ┌───────────────────────────────────────┐ │
│ │            │ │  헤더 (페이지 제목 + 설명)             │ │
│ │  사이드바   │ ├───────────────────────────────────────┤ │
│ │            │ │                                       │ │
│ │  - 메뉴    │ │                                       │ │
│ │  - 연결상태│ │         메인 콘텐츠 영역               │ │
│ │  - 진행패널│ │                                       │ │
│ │            │ │                                       │ │
│ │  [설정]    │ │                                       │ │
│ └────────────┘ └───────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```

### 사이드바

- **메뉴**: 대시보드, 파일 탐색기, 폴더 설정, 동기화 로그, 통계, 마이그레이션, 설정
- **연결 상태 표시**: 외부웹하드 / 자체웹하드 각각의 연결 상태를 색상 점으로 표시
  - 초록: 연결됨
  - 빨강: 연결 안 됨
- **동기화 진행 패널**: 전체 동기화 진행 중일 때만 표시되며, 현재 단계/진행률/파일명/속도/남은 시간을 보여줌
- **로그 뱃지**: 동기화 로그 메뉴 옆에 오늘 실패 건수가 빨간 뱃지로 표시 (최대 99+)
- **접기/펼치기**: 좌측 상단 화살표 버튼으로 사이드바를 축소(64px)/확장(220px) 전환 가능

## 3. 페이지별 상세 설명

### 3-1. 대시보드

> 동기화 현황을 실시간으로 모니터링하는 메인 화면

```
┌─────────────────────────────────────────────────┐
│  동기화 상태 카드                                │
│  ● 동기화 중 (또는 대기/일시정지/오류/연결끊김)   │
│  [진행 바 - 전체동기화 중일 때만]                 │
│  [실패 N건 재시도] [전체 동기화] [일시중지/재개]   │
├─────────────────────────────────────────────────┤
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐           │
│  │전체  │ │성공  │ │실패  │ │전송량│           │
│  │ 50건 │ │ 48건 │ │ 2건  │ │1.2GB│           │
│  └──────┘ └──────┘ └──────┘ └──────┘           │
├────────────────────┬────────────────────────────┤
│  현재 전송 중       │  최근 동기화 파일           │
│  ↓ 견적서.pdf  45% │  ✓ 도면_A.dwg   2분 전     │
│  ↑ 계약서.pdf  80% │  ✓ 사양서.xlsx  5분 전     │
│                    │  ✗ 보고서.pdf   10분 전    │
└────────────────────┴────────────────────────────┘
```

**주요 기능:**

| 영역 | 설명 |
|------|------|
| 상태 카드 | 현재 동기화 엔진 상태를 색상+아이콘으로 표시. 전체동기화 중이면 단계명과 진행 바가 추가로 표시됨 |
| 오늘의 통계 | 전체/성공/실패/전송량 4개 카드. 오늘 자정 기준으로 집계 |
| 현재 전송 중 | 다운로드(↓) 또는 업로드(↑) 진행 중인 파일 최대 5건. 파일명, 속도, 진행률 표시 |
| 최근 동기화 | 최근 처리된 파일 최대 20건. 상태 아이콘(성공/실패), 파일명, 폴더, 크기, 시간 표시 |

**버튼 동작:**

| 버튼 | 동작 |
|------|------|
| `실패 N건 재시도` | 실패 건수가 1건 이상일 때만 활성화. 확인 다이얼로그 후 실패 파일을 재시도 |
| `전체 동기화` | 확인 다이얼로그 후 활성화된 모든 폴더의 전체 파일을 스캔하여 동기화 시작 |
| `일시중지` / `재개` | 동기화 엔진을 일시정지하거나 다시 시작 |

### 3-2. 파일 탐색기

> 폴더 트리 탐색, 파일 목록 조회, 개별 파일 상세 확인 및 재시도

```
┌─────────────────────────────────────────────────┐
│  [🔍 파일명 검색...]                    [새로고침]│
├──────────────┬──────────────────────────────────┤
│  폴더 트리    │  파일 목록                       │
│              │                                  │
│  ▶ 회사A     │  상태  파일명      크기    시간   │
│  ▼ 회사B     │  ───────────────────────────────  │
│    └ 하위1   │  ✓   도면.dwg    2.1MB  09:30   │
│  ▶ 회사C     │  ✗   견적서.pdf   500KB  09:25   │
│              │  ⏳  사양서.xlsx   1.2MB  진행중   │
│              │                                  │
│              │          [◀ 1 / 3 ▶]             │
├──────────────┴──────────────────────────────────┤
│  선택된 파일 상세                                 │
│  파일명: 견적서.pdf | 상태: 실패 | 오류: 연결 시간초과│
│  [지금 동기화]  [로그 보기]                       │
└─────────────────────────────────────────────────┘
```

**주요 기능:**

| 기능 | 설명 |
|------|------|
| 폴더 트리 | 좌측에 계층형 폴더 구조 표시. 폴더 클릭 시 해당 폴더의 파일만 필터링. 비활성 폴더는 흐리게 표시 |
| 파일 목록 | 상태 아이콘 / 파일명 / 크기 / 동기화 시간 / 상태 뱃지 컬럼. 헤더 클릭으로 정렬 변경 (오름차순↔내림차순) |
| 검색 | 파일명 키워드 검색. 입력 후 0.3초 대기 후 자동 검색 (디바운스) |
| 페이지네이션 | 1페이지당 50건, 이전/다음 버튼으로 이동 |
| 파일 상세 | 파일 행 클릭 시 하단에 상세 정보 패널 표시. 실패 파일은 `지금 동기화` 버튼으로 즉시 재시도 가능. `로그 보기` 클릭 시 동기화 로그 페이지로 이동 |

### 3-3. 폴더 설정

> 동기화 대상 폴더를 관리하고, LGU+ 폴더를 자동 검색하여 등록

```
┌─────────────────────────────────────────────────┐
│  폴더 설정           [LGU+ 폴더 스캔] [새로고침] │
│                                                 │
│  [스캔 완료: 2개의 새 폴더가 발견되었습니다]      │
│                                                 │
│  ☑  📁 회사A    파일 120건   마지막: 5분 전  ████│
│  ☑  📁 회사B    파일 85건    마지막: 1시간 전 ███ │
│  ☐  📁 회사C    파일 30건    마지막: -        ░░░│
│                                                 │
│  ─────────────────────────────────────────────  │
│  선택됨: 2 / 전체 3   │  총 파일: 205건          │
└─────────────────────────────────────────────────┘
```

**주요 기능:**

| 기능 | 설명 |
|------|------|
| 체크박스 | 개별 폴더의 동기화 활성/비활성을 토글. 비활성 폴더는 동기화 대상에서 제외됨 |
| LGU+ 폴더 스캔 | LGU+ 웹하드의 `올리기전용` 하위 폴더를 자동으로 탐색하여 신규 폴더를 DB에 등록. 동시에 자체웹하드에도 대응 폴더를 자동 생성 |
| 상태 바 | 각 폴더의 동기화 진행 상태를 미니 프로그레스 바로 시각화 |
| 하단 요약 | 활성 폴더 수 / 전체 폴더 수, 활성 폴더의 총 파일 수 표시 |

**LGU+ 폴더 구조:**

```
HOME (게스트 루트)
└── 올리기전용
    ├── 회사A    ← 동기화 대상으로 등록됨
    ├── 회사B    ← 동기화 대상으로 등록됨
    └── 회사C    ← 동기화 대상으로 등록됨
```

├─────────────────────────────────────────────────────┤
│  시간           레벨    메시지                        │
│  ─────────────────────────────────────────────────  │
│  09:30:15.123   INFO   파일 동기화 완료: 도면.dwg    │
│  09:25:02.456   ERROR  업로드 실패: 견적서.pdf [재시도]│
│  ▼ 09:25:02.456 (펼침)                              │

**주요 기능:**

| 기능 | 설명 |
|------|------|

### 3-6. 마이그레이션

> 외부웹하드의 기존 파일을 자체웹하드로 일괄 이전하는 기능

마이그레이션은 4단계 상태 머신으로 동작합니다:

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│          │     │          │     │          │     │          │
│   대기   │────▶│  스캔중  │────▶│  선택    │────▶│  진행중  │────▶ 완료
│  (idle)  │     │(scanning)│     │(selecting│     │(migrating│
│          │     │          │     │          │     │          │
└──────────┘     └──────────┘     └──┬───────┘     └──────────┘
                                     │     ▲
                                     │     │
                                     └─────┘
                                    다시 스캔
```

**각 단계별 화면:**

| 단계 | 화면 내용 |
|------|----------|
| **대기** | 중앙에 안내 아이콘과 설명 텍스트. `폴더 스캔 시작` 버튼 |
| **스캔 중** | 로딩 스피너와 "스캔 중..." 메시지 |
| **선택** | 발견된 폴더 목록을 테이블로 표시. 각 폴더별 전체/완료/남은 파일 수 확인 가능. 전체 선택/해제 토글, 개별 체크박스로 대상 폴더 선택 후 `마이그레이션 시작` 클릭 |
| **진행 중** | 진행 스피너, 선택 폴더/파일 수, 현재 처리 중인 파일명 표시. **"앱을 종료하지 마세요"** 경고 |
| **완료** | 결과 4개 카드 (스캔된 폴더 / 신규 폴더 / 성공 / 실패), 총 파일 수, 소요 시간. `처음으로 돌아가기` 버튼 |

**마이그레이션 vs 전체 동기화 차이:**

| 항목 | 전체 동기화 | 마이그레이션 |
|------|-----------|-------------|
| 용도 | 일상적인 누락 파일 보정 | 초기 대량 이전 |
| 폴더 범위 | 활성화된 모든 폴더 | 사용자가 선택한 폴더만 |
| 신규 폴더 | 기존 등록된 폴더만 | 스캔하여 신규 폴더도 자동 등록 |
| UI | 대시보드에서 버튼 하나로 실행 | 전용 페이지에서 단계별 진행 |

## 4. 전체 동기화 작동 원리

대시보드에서 `전체 동기화` 버튼을 클릭하면 다음 과정이 실행됩니다.

### 전체 흐름도

```
사용자: [전체 동기화] 클릭
    │
    ▼
확인 다이얼로그 표시
    │ [확인]
    ▼
┌─────────────────────────────────────────────┐
│ Phase 1: 폴더 목록 조회                      │
│                                             │
│  DB에서 활성화된(enabled=true) 폴더 목록 조회  │
│  예: [회사A, 회사B] (회사C는 비활성이므로 제외) │
└───────────────────┬─────────────────────────┘
                    │
                    ▼ (각 폴더에 대해 순차 처리)
┌─────────────────────────────────────────────┐
│ Phase 2: 파일 스캔                           │
│                                             │
│  LGU+ API로 해당 폴더의 전체 파일 목록 조회    │
│  (페이지네이션으로 전체 조회)                   │
│  예: 회사A → [도면.dwg, 견적서.pdf, ...] 50건 │
└───────────────────┬─────────────────────────┘
                    │
                    ▼ (각 파일에 대해)
┌─────────────────────────────────────────────┐
│ Phase 3: 중복 확인                           │
│                                             │
│  DB에서 해당 파일이 이미 동기화되었는지 확인    │
│  ├─ 이미 completed → SKIP (다음 파일로)      │
│  └─ 신규 또는 실패 → DB에 저장 후 동기화 진행  │
└───────────────────┬─────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│ Phase 4: 다운로드 (LGU+ → 로컬)             │
│                                             │
│  1. LGU+ API에서 다운로드 세션/URL 획득       │
│  2. 인증 파라미터 포함한 URL로 파일 다운로드    │
│  3. 파일 크기 검증 (불일치 시 오류)            │
│  4. 로컬 임시 폴더에 저장                     │
│                                             │
│  ※ 실패 시 최대 3회 재시도 (지수 백오프)       │
│  ※ 5회 연속 실패 시 서킷 브레이커 OPEN        │
└───────────────────┬─────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│ Phase 5: 업로드 (로컬 → 자체웹하드)           │
│                                             │
│  1. 자체웹하드에 대상 폴더 존재 여부 확인       │
│     └─ 없으면 자동 생성 (올리기전용/회사A)     │
│  2. Presigned URL 발급 요청                  │
│     └─ 이미 업로드된 파일이면 SKIP            │
│  3. R2(Cloudflare)에 직접 업로드              │
│  4. 메타데이터를 자체웹하드 DB에 등록           │
│                                             │
│  ※ 실패 시 최대 3회 재시도 (지수 백오프)       │
│  ※ API Key 오류는 재시도 없이 즉시 실패        │
└───────────────────┬─────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│ Phase 6: 완료 처리                           │
│                                             │
│  1. DB 상태를 'completed'로 갱신              │
│  2. 일별 통계 업데이트 (성공 +1, 바이트 누적)  │
│  3. UI에 완료 이벤트 전파                     │
└─────────────────────────────────────────────┘
```

### 파일 1건의 상태 전이

```
pending → downloading → uploading → completed
              │              │
              ▼              ▼
          dl_failed      ul_failed
              │              │
              └──────────────┘
                     │
                     ▼
              재시도 시 pending으로 복귀
```

| 시도 | 지연 |
|------|------|
| 1차 재시도 | 1초 |
| 2차 재시도 | 2초 |
| 3차 재시도 | 4초 |
| 최대 지연 | 10초 |

## 5. 실시간 폴링 동기화

프로그램 시작 후 엔진이 가동되면, 설정된 간격(기본 5초)마다 자동으로 새 파일을 감지합니다.

### 폴링 흐름도

```
┌──────────────────────────────────────────────┐
│                5초마다 반복                    │
│                                              │
│  LGU+ 업로드 히스토리 API 호출                 │
│           │                                  │
│           ▼                                  │
│  마지막 체크포인트(history_no) 이후의          │
│  신규 항목만 필터링                            │
│           │                                  │
│      ┌────┴────┐                             │
│      │         │                             │
│   신규 없음   신규 발견                        │
│      │         │                             │
│   대기        ▼                              │
│           체크포인트 갱신 (DB 저장)             │
│                │                             │
│                ▼                             │
│           SyncEngine에 파일 목록 전달          │
│                │                             │
│                ▼                             │
│           각 파일 동기화 시작                   │
│           (다운로드 → 업로드 → 완료)           │
└──────────────────────────────────────────────┘
```

### 체크포인트 메커니즘

- LGU+ 업로드 히스토리의 각 항목에는 고유 `historyNo`가 있습니다
- 마지막으로 처리한 `historyNo`를 DB에 저장(체크포인트)합니다
- 다음 폴링 시 체크포인트보다 큰 번호의 항목만 처리합니다
- **프로그램 재시작 후에도 중복 처리를 방지**합니다

## 7. 오류 처리 및 재시도

### 서킷 브레이커

연속적인 오류가 발생할 때 시스템을 보호하는 안전장치입니다.

```
     정상 동작               연속 5회 실패          10초 경과
┌──────────┐          ┌──────────┐          ┌──────────┐
│  CLOSED  │ ───────▶ │   OPEN   │ ───────▶ │HALF_OPEN │
│ (정상)   │          │ (차단)   │          │ (시험)   │
└──────────┘          └──────────┘          └────┬─────┘
     ▲                      ▲                    │
     │                      │               성공? │ 실패?
     │                      └────────────────────┘
     │                                      │
     └──────────────────────────────────────┘
```

| 상태 | 동작 |
|------|------|
| **CLOSED** (정상) | 모든 요청을 정상 처리 |
| **OPEN** (차단) | 5회 연속 실패 후 활성화. 모든 요청을 즉시 거부하여 외부 서비스 부하 방지 |
| **HALF_OPEN** (시험) | 10초 후 1건만 시험 요청. 성공하면 CLOSED로, 실패하면 다시 OPEN으로 |

두 개의 독립된 서킷이 운영됩니다:
- `lguplus-download`: LGU+ 다운로드 회로
- `webhard-upload`: 자체웹하드 업로드 회로

### 재시도 불가능한 오류

다음 오류는 재시도 없이 즉시 실패 처리됩니다:

| 오류 | 설명 |
|------|------|
| API Key 인증 실패 | 자체웹하드 API Key가 잘못된 경우 |
| 세션 재갱신 실패 | LGU+ 재로그인이 2회 연속 실패한 경우 |

### 실패 파일 재시도 방법

| 방법 | 위치 | 설명 |
|------|------|------|
| 대시보드 일괄 재시도 | 대시보드 > `실패 N건 재시도` | 모든 실패 파일을 일괄 재시도 |
| 파일 탐색기 개별 재시도 | 파일 탐색기 > 파일 선택 > `지금 동기화` | 특정 파일 1건만 재시도 |
| 로그 뷰어 개별 재시도 | 동기화 로그 > ERROR 로그의 `재시도` | 해당 오류의 파일을 재시도 |
